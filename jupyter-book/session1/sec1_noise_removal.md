# 노이즈 제거 — 신호에서 의미를 꺼내는 법

---

## 1-1. 문제 제기

- CNC 밀링머신 진동 센서가 수집하는 데이터에는 **여러 신호가 뒤섞여** 있음
- 신호 = 시간에 따라 변하는 물리량 (Think DSP 1장)
- CNC 진동 센서도 **트랜스듀서(변환기)**: 물리적 진동을 전기 신호로 변환

```
[실제 CNC 진동 데이터에 섞인 성분들]

절삭 신호     ──── 우리가 원하는 것 (공구-소재 접촉 정보)
스핀들 노이즈 ──── 회전체의 기계적 진동 (고주파, 일정한 패턴)
채터링       ──── 공구 떨림 (불규칙 고주파, 가공 불량의 전조)
전기 간섭    ──── 모터/인버터에서 오는 전자기 노이즈 (특정 주파수에 집중)
```

- Raw plot에서는 이 성분들을 **육안으로 구분 불가능**
- 오케스트라 녹음에 비유: 전체 소리만 듣고 각 악기의 음을 구분할 수 없는 것과 동일
- **핵심 질문**: 우리가 원하는 신호만 꺼낼 수 있을까?

---

## 1-2. 이론

### ① FFT로 먼저 보면 어떨까?

> Think DSP Ch.7 DFT / Ch.3 Non-Periodic Signals

**FFT(Fast Fourier Transform)** 는 신호를 주파수 성분으로 분해하는 알고리즘입니다. Think DSP 교재는 이를 **스펙트럼 분해(Spectral Decomposition)** 라 부르며 책 전체에서 가장 중요한 아이디어로 강조합니다. 시간에 따라 기복하는 파형을 "어떤 주파수 성분이 얼마나 들어 있는가"로 바꿔 보여 주기 때문입니다. 수학적으로는 "임의의 신호는 서로 다른 주파수를 갖는 사인파들의 합으로 표현될 수 있다"는 푸리에 정리가 그 근거입니다.

```{mermaid}
flowchart LR
    A["시간 영역 신호 x(t)"] --> B["FFT"]
    B --> C["주파수 스펙트럼 X(f)"]
```

FFT의 장점은 분명합니다. "어떤 주파수가 얼마나 있는가"를 한눈에 파악할 수 있다는 것이죠. 그러나 결정적인 단점이 하나 있습니다. 바로 **"그 주파수가 언제 발생했는가"를 알 수 없다**는 점입니다. 예를 들어 CNC 가공 중 채터링이 시작 30초에 발생했는지 50초에 발생했는지는 FFT만으로 구분할 수 없습니다. 신호 전체를 하나의 스펙트럼으로 압축해 버리기 때문에, 시간 정보가 사라지는 것입니다. 이런 문제를 다루려면 시간과 주파수를 동시에 보는 분석이 필요합니다.

```{admonition} 핵심 한계
:class: important

FFT는 신호 전체를 하나의 스펙트럼으로 압축하므로, **시간에 따라 변하는(비정상 신호, non-stationary) 데이터에는 부적합**합니다. 현실의 거의 모든 신호가 여기 해당됩니다.
```

#### 스펙트럼 분해 — 신호의 주파수 구조

교재의 예시를 따라가 보겠습니다. 종을 치면 정현파에 가까운 깨끗한 신호가 나오지만, 바이올린을 연주하면 훨씬 복잡한 복합 파형이 나옵니다. 이 바이올린 신호를 FFT로 분석하면 여러 개의 피크가 나타납니다.

```{figure} ../../lecture/images/s1_1_img64.png
:alt: Figure 1-1. 종(Bell) 녹음 신호 — 주기적 정현파
:width: 67%

Figure 1-1. 종(Bell) 녹음 신호 — 주기적 정현파
```

- 종소리 녹음을 **시간 영역**에서 본 파형으로, 진폭이 거의 일정한 매끈한 정현파가 규칙적으로 반복됨
- 하나의 기본 주파수가 지배적인 **주기 신호(periodic signal)** 의 전형 → 성분이 단순해 분석이 쉬움
- 뒤에 나올 복잡한 바이올린 신호와 대비되는 "깨끗한 신호"의 기준 예시

```{figure} ../../lecture/images/s1_1_img32.png
:alt: Figure 1-2. 바이올린 녹음 신호 — 복합 파형
:width: 67%

Figure 1-2. 바이올린 녹음 신호 — 복합 파형
```

- 같은 시간 영역이지만 여러 정현파가 겹쳐 파형이 복잡하게 일그러져 있음
- 하나의 음(피치)이라도 **기본 주파수 + 여러 고조파**가 섞여 있음을 시사
- 여러 성분이 뒤섞인 실제 CNC 진동 신호와 닮은 형태 → "육안 구분 불가" 문제의 축소판

```{figure} ../../lecture/images/s1_1_img74.png
:alt: Figure 1-3. 바이올린 신호의 스펙트럼 — 기본 주파수와 고조파
:width: 67%

Figure 1-3. 바이올린 신호의 스펙트럼 — 기본 주파수와 고조파
```

- 바이올린 신호를 **FFT로 분해**한 주파수 스펙트럼 (가로축: 주파수 Hz, 세로축: 진폭)
- 가장 왼쪽의 큰 피크가 **기본 주파수(f₀)**, 그 정수배 위치의 피크들이 **고조파(harmonics)**
- 시간 영역에서 복잡해 보이던 신호가 주파수 영역에선 몇 개의 또렷한 피크로 정리됨 → FFT의 힘

스펙트럼에서 가장 낮은 주파수 성분을 **기본 주파수(fundamental frequency)**, 그 정수배 위치에 나타나는 성분들을 **고조파(harmonics)** 라고 부릅니다. 예컨대 A4 음(440Hz)의 바이올린 신호라면 기본이 440Hz, 2차 고조파가 880Hz(A5, 한 옥타브 위), 3차 고조파가 1320Hz(E6, 완전 5도 위)로 나타납니다.

CNC 진동 데이터에도 같은 원리가 적용됩니다. 절삭 신호는 특정 주파수에 피크를 만들고, 스핀들 노이즈는 고주파 대역에 규칙적으로 퍼지며, 채터링은 불규칙한 고주파로 나타납니다. 한편 노이즈 분석에서는 진폭 대신 **파워(Power)**, 즉 진폭의 제곱을 주로 사용하는데, 에너지 측면에서 신호의 강도를 더 잘 나타내기 때문입니다.

### Spectrogram — FFT의 시간 정보 상실 문제 해결

FFT가 시간 정보를 잃는 문제는 **Spectrogram(스펙트로그램)** 으로 해결할 수 있습니다. 신호를 짧은 구간으로 나누어 각 구간마다 스펙트럼을 계산한 뒤, 그 결과를 시간-주파수 2차원 플롯으로 그리는 방식입니다. 이렇게 구간별로 푸리에 변환을 수행하는 기법을 **STFT(Short-Time Fourier Transform)** 라 부릅니다. 구체적으로는 신호를 Hamming 윈도우로 짧게 자른 뒤 각 구간에 FFT를 적용하고, 그 결과를 시간-주파수 2차원 맵으로 이어 붙입니다.

```
[Spectrogram 구조]

    주파수 ↑
           │  ■     ■        ← 고주파 성분이 특정 시간에만 나타남
           │ ■■   ■■■       ← 시간에 따른 주파수 변화가 보임
           │■■■ ■■■■■■      ← FFT 단일 스펙트럼에서는 놓치는 정보
           └────────────→ 시간
```

주파수가 시간에 따라 변하는 처프(Chirp) 신호가 좋은 예입니다. 처프의 스펙트로그램을 보면 주파수가 시간에 따라 어떻게 변하는지 선명하게 드러나는데, 이는 단일 FFT 스펙트럼으로는 결코 볼 수 없는 정보입니다.

```{figure} ../../lecture/images/s1_1_img49.png
:alt: Figure 3-3. 처프(Chirp)의 스펙트로그램
:width: 67%

Figure 3-3. 처프(Chirp)의 스펙트로그램
```

- 주파수가 시간에 따라 변하는 **처프 신호**를 시간-주파수 2차원으로 시각화 (가로축: 시간, 세로축: 주파수, 색/밝기: 세기)
- 시간이 흐르며 주파수가 비스듬히 이동하는 선이 보임 → "언제 어떤 주파수가 나타나는가"를 포착
- 단일 FFT로는 절대 알 수 없는 정보 → **STFT/스펙트로그램의 핵심 가치**

```{figure} ../../lecture/images/s1_1_img41.png
:alt: Figure 3-5. 정현파, Hamming 윈도우, 곱셈 결과
:width: 67%

Figure 3-5. 정현파, Hamming 윈도우, 곱셈 결과
```

- STFT의 한 구간 처리 과정: **원본 정현파 조각 × Hamming 윈도우 = 양끝이 0으로 부드럽게 수렴**한 신호
- 윈도우를 곱하는 이유는 구간 경계의 급격한 불연속이 만드는 **누설(spectral leakage)** 을 줄이기 위함
- 각 구간마다 이렇게 윈도우를 씌운 뒤 FFT를 수행하는 것이 STFT의 한 스텝

```{figure} ../../lecture/images/s1_1_img66.png
:alt: Figure 3-6. 겹치는 Hamming 윈도우
:width: 67%

Figure 3-6. 겹치는 Hamming 윈도우
```

- 연속된 Hamming 윈도우들이 서로 **겹치며(overlap)** 신호 전체를 빠짐없이 덮는 모습
- 겹침 덕분에 윈도우 양끝에서 감쇠된 정보가 옆 구간에서 보완되어 시간 연속성이 유지됨
- 스펙트로그램이 구간들을 어떻게 이어 붙여 계산하는지를 설명

```{admonition} Gabor Limit (가보르 한계)
:class: important

구간을 짧게 자르면 시간 해상도는 좋아지지만 주파수 해상도가 뭉개지고, 길게 자르면 그 반대가 됩니다. 시간과 주파수 해상도를 동시에 완벽하게 얻을 수는 없는데, 이는 물리학의 불확정성 원리와 같은 종류의 근본적 트레이드오프입니다. 뒤에서 배울 **Wavelet Transform** 은 주파수에 따라 윈도우 크기를 자동으로 조절(저주파에는 긴 윈도우, 고주파에는 짧은 윈도우)하여 이 Gabor Limit을 우회합니다.
```

### ② Wavelet Transform의 직관

> Think DSP Ch.3 / Ch.8

Wavelet Transform은 한마디로 **"돋보기의 배율을 바꿔 가며 신호를 훑는 것"** 에 비유할 수 있습니다. 큰 스케일(저주파 돋보기)로는 절삭 패턴 같은 신호의 전체적인 추세를 보고, 중간 스케일로는 스핀들 진동 같은 중간 주파수 성분을, 작은 스케일(고주파 돋보기)로는 채터링이나 전기 노이즈 같은 빠른 변화를 봅니다. 그래서 분해 결과를 보면 레벨 1(고주파)에는 전기 간섭과 채터링이, 레벨 2(중주파)에는 스핀들 노이즈가 들어가 제거 대상이 되고, 레벨 3(저주파)의 절삭 신호와 근사(Approximation) 성분의 장기 추세는 보존 대상이 됩니다.

```{mermaid}
flowchart TB
    subgraph "Wavelet 분석 결과"
        L1["레벨 1 (고주파) — 전기 간섭 + 채터링 → 제거 대상"]
        L2["레벨 2 (중주파) — 스핀들 노이즈 → 제거 대상"]
        L3["레벨 3 (저주파) — 절삭 신호 → 보존 대상"]
        L4["Approximation — 장기 추세 → 보존 대상"]
    end
```

FFT와의 결정적 차이는 여기에 있습니다. FFT는 전체 시간을 하나로 압축하지만, Wavelet은 **주파수 정보와 시간 위치 정보를 동시에** 유지합니다. 덕분에 어떤 성분이 노이즈인지뿐 아니라 그것이 언제 나타나는지까지 알 수 있습니다. 이제 어떤 성분이 노이즈인지 알았으니, 이를 제거하는 방법을 살펴봅시다.

### 노이즈의 종류와 특성

> Think DSP Ch.4 Noise

노이즈를 제거하려면 먼저 그 정체를 알아야 합니다. 교재는 노이즈를 두 가지 의미로 정의합니다. 하나는 "원하지 않는 신호"라는 일상적 의미이고, 다른 하나는 "다양한 주파수 성분을 포함하여 조화 구조가 없는 신호"라는 신호 처리적 의미인데, 여기서 다루는 것은 두 번째 의미입니다.

```{figure} ../../lecture/images/s1_1_img28.png
:alt: Figure 4-1. 비상관 균일 잡음의 파형
:width: 67%

Figure 4-1. 비상관 균일 잡음의 파형
```

- 시간 영역에서 본 **백색(균일) 잡음** — 인접 샘플 간 상관이 없어 완전히 무작위로 출렁임
- 어떤 규칙·주기도 없는 "조화 구조가 없는 신호"의 전형 → 신호 처리에서 말하는 노이즈의 정의
- 제거 대상 노이즈가 시간 영역에서 어떻게 생겼는지 보여주는 출발점

```{figure} ../../lecture/images/s1_1_img04.png
:alt: Figure 4-2. 잡음의 파워 스펙트럼
:width: 67%

Figure 4-2. 잡음의 파워 스펙트럼
```

- 같은 잡음을 **주파수 영역**에서 본 파워 스펙트럼 — 특정 피크 없이 전 주파수에 고르게 퍼짐
- 백색 잡음은 모든 주파수에 비슷한 파워 → 스펙트럼이 대체로 **평평**
- 신호(특정 피크)와 노이즈(넓게 퍼짐)를 주파수 영역에서 구분하는 근거

```{figure} ../../lecture/images/s1_1_img33.png
:alt: Figure 4-7. 백색·분홍·적색 잡음 스펙트럼 비교
:width: 67%

Figure 4-7. 백색·분홍·적색 잡음 스펙트럼 비교
```

- 세 가지 잡음의 파워 스펙트럼을 **로그 축**에서 비교 — 기울기로 종류가 구분됨
- 백색=평평(P 일정), 분홍=완만한 하강(P∝1/f), 적색/갈색=가파른 하강(P∝1/f²)
- 노이즈마다 주파수 분포가 다름 → **어느 대역을 눌러야 하는지** 제거 전략이 달라지는 이유

노이즈는 파워가 주파수에 따라 어떻게 분포하는가로 분류합니다. **백색 잡음(White Noise)** 은 모든 주파수에서 파워가 일정해 스펙트럼이 평평하며, 전기 간섭이나 센서 열잡음이 여기 해당합니다. **갈색 잡음(Brownian/Red Noise)** 은 파워가 주파수의 제곱에 반비례(P ∝ 1/f²)해 저주파에 에너지가 몰리며, 기계적 드리프트나 온도 변화에 의한 센서 출력 이동이 그 예입니다. **분홍 잡음(Pink Noise)** 은 P ∝ 1/f로 백색과 갈색의 중간이고, **가우시안 잡음(Gaussian Noise)** 은 값이 정규분포를 따르는 무작위 잡음으로 곧 배울 Universal Threshold의 이론적 기반이 됩니다.

```{admonition} 제조 데이터에서의 의미
:class: important

CNC 진동 노이즈는 대부분 **백색 잡음 + 가우시안 잡음의 혼합**. 고주파 계수(cD1)에서 σ를 추정하여 Threshold 계산에 사용합니다.
```

### ③ Hard/Soft Threshold: 어떤 성분을 버릴 것인가

> Think DSP Ch.8 Filtering and Convolution / Ch.4 Noise

Wavelet으로 신호를 분해한 다음에는 각 주파수 대역의 계수를 보고 그것이 노이즈인지 아닌지 판단해야 합니다. 이때 사용하는 것이 **Threshold(임계값)** 기반 필터링이며, 주파수 영역에서 작동합니다.

**Hard Threshold** 는 계수의 절댓값이 임계값 T보다 작으면 0으로 잘라내고 크면 그대로 두는 방식으로, 가위로 싹 자르듯 깔끔합니다. **Soft Threshold** 는 T보다 작으면 0으로 만들고 크면 크기를 T만큼 줄여 부드럽게 처리합니다. Hard는 강한 신호를 잘 보존하지만 경계에서 불연속이 생길 수 있고, Soft는 신호가 약간 왜곡되는 대신 더 매끄러운 결과를 줍니다. 다양한 노이즈가 섞인 제조 데이터 실무에서는 보통 **Soft Threshold가 더 안정적**입니다.

```{admonition} 핵심 — Universal Threshold
:class: important

**임계값 T**:

$$T = \sigma \times \sqrt{2 \times \ln(N)}$$

- σ: 노이즈 표준편차 → 고주파 계수 **cD1**에서 추정 (노이즈가 압도적으로 많다는 가정)
- N: 신호 샘플 수
- 가우시안 잡음 가정이 중요
- Donoho & Johnstone, 1994
```

- 이동평균(저역통과)이 급격한 변화(고주파 성분)를 제거함을 시각적으로 확인
- Wavelet에서 고주파 계수를 threshold로 줄이는 것과 **같은 원리**

```{figure} ../../lecture/images/s1_1_img52.png
:alt: Figure 8-7. 가우시안 평활화 스펙트럼 비율
:width: 67%

```

- 가우시안 평활 필터의 **주파수 응답**(입력 대비 출력의 비율)을 주파수에 따라 표시
- 저주파는 비율 ≈ 1(통과), 고주파로 갈수록 비율 ↓(감쇠) → 전형적인 **저역통과** 특성
- "시간 영역의 합성곱 = 주파수 영역의 곱셈"(합성곱 정리)을 수치로 보여줌

**합성곱 정리(Convolution Theorem)**: 시간 영역에서의 합성곱 = 주파수 영역에서의 곱셈

$$\text{DFT}(f * g) = \text{DFT}(f) \times \text{DFT}(g)$$

합성곱 정리는 FFT가 빠른 이유이기도 합니다. 시간 영역에서 직접 합성곱을 계산하면 $O(N^2)$이지만 주파수 영역의 곱셈으로 바꾸면 $O(N \log N)$으로 줄어듭니다. Wavelet threshold도 같은 원리로 작동합니다. 먼저 Wavelet 분해로 신호를 주파수 영역으로 옮기고, 계수에 threshold를 적용해 특정 주파수 성분을 감쇠시킨 뒤, 역 Wavelet 변환으로 다시 시간 영역으로 복원합니다.

### ④ pywt 라이브러리 핵심 함수 3개

> Think DSP Ch.8 — pywt의 분해/재합성 대응 이론

이론은 여기까지입니다. 실제 파이썬에서는 **pywt** 라이브러리를 사용하는데, 그 흐름은 레고 블록을 분해하고 각 파트를 점검한 뒤 다시 조립하는 것과 같은 3단계입니다. `wavedec`으로 신호를 여러 레벨로 분해하고, 고주파 계수에 threshold를 적용해 노이즈를 0에 가깝게 만든 뒤, `waverec`으로 다시 재합성합니다.

```python
import pywt, numpy as np

# 1. 사용 가능한 wavelet 종류 확인
print(pywt.wavelist(kind='discrete'))  # db, sym, haar 등

# 2. 다중 레벨 분해 (DWT)
coeffs = pywt.wavedec(signal, wavelet='db4', level=5)
# coeffs = [cA5, cD5, cD4, cD3, cD2, cD1]
#            근사  디테일5  ...  디테일1(최고주파)

# 3. 재합성 (역 DWT)
reconstructed = pywt.waverec(coeffs_modified, wavelet='db4')
```

```{mermaid}
flowchart TB
    A["원본 신호"] --> B["pywt.wavedec() — 분해"]
    B --> C["계수 배열 cA, cD5, cD4, cD3, cD2, cD1"]
    C --> D["threshold 적용 — 노이즈 계수를 0에 가깝게"]
    D --> E["수정된 계수 배열"]
    E --> F["pywt.waverec() — 재합성"]
    F --> G["노이즈 제거된 신호"]
```

```{admonition} 주의
:class: warning

분해할 때 쓴 wavelet 이름(예: `db4`)과 재합성할 때 쓴 이름이 **같아야** 제대로 복원됩니다.
```

## 핵심 요약

1. **FFT**는 주파수 성분은 보여주지만 **시간 정보를 잃는다**
2. **Wavelet Transform**은 **시간과 주파수 정보를 동시에 유지**한다
3. **pywt의 wavedec → threshold → waverec** 3단계 파이프라인으로 노이즈를 제거할 수 있다
