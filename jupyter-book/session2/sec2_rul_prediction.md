# RUL 예측 — 얼마나 버틸 수 있는가

---

## 2-1. 문제 제기

Autoencoder는 "지금 이상한가?"에 답하는 이진 판정 모델이었습니다. 이번에는 한 단계 넘어가 **RUL(Remaining Useful Life, 잔여 유효 수명)** 을 예측하는 회귀 문제를 다룹니다. 다시 말해 "앞으로 얼마나 더 쓸 수 있는가?"에 답하는 것이 목표입니다.

```{mermaid}
flowchart LR
    A["① 사후보전\n고장 → 수리\n비용: 최대\n위험: 최대"] --> B["② 예방보전\n일정 주기마다 교체\n비용: 중간\n위험: 낮음"]
    B --> C["③ 예지보전 ← 목표\nRUL 예측 기반 교체\n비용: 최소\n위험: 최저"]
```

```{figure} ../../lecture/images/s2_2_img01.png
:alt: 설비 유지보수 전략 비교
:width: 67%

설비 유지보수 전략 비교
```

설비 유지보수에는 세 가지 전략이 있습니다. **사후보전** 은 고장 나면 고치는 방식으로 긴급 수리 비용과 생산 중단이라는 큰 대가를 치릅니다. **예방보전** 은 일정 주기로 교체하지만, 아직 쓸 수 있는 부품을 버리는 낭비가 생깁니다. 30일마다 교체하면서 실제로는 50일까지 쓸 수 있는 부품이 그 예입니다. **예지보전** 은 RUL 예측을 바탕으로 교체 시점을 결정해 필요한 순간에 정확하게 교체합니다. 오른쪽으로 갈수록 비용과 위험이 동시에 낮아집니다.

```
엔진이 총 300 사이클 후 고장난다면:

사이클   1: RUL = 299
사이클 100: RUL = 200
사이클 250: RUL = 50   ← 이 시점에 교체 결정
사이클 300: RUL = 0    ← 고장
```

핵심 질문: 현재 센서 데이터만으로 RUL을 예측할 수 있는가?

## 2-2. 이론

### ① 왜 일반 회귀가 아닌 LSTM인가

RUL 예측은 숫자를 맞추는 회귀 문제. 그런데 **왜 선형 회귀나 MLP로는 부족할까?**

답은 **순서가 중요하기 때문** 입니다. 예를 들어 온도 300도, 진동 0.02라는 똑같은 센서 값이라도 사이클 50에서 나오면 정상이지만 사이클 250에서 나오면 마모 징후일 수 있습니다. 즉 "지금 값" 하나가 아니라 "값이 변해온 패턴"이 RUL을 좌우합니다.

```{figure} ../../lecture/images/s2_2_img11.png
:alt: 순차 데이터: MLP vs RNN 비교
:width: 67%

순차 데이터: MLP vs RNN 비교
```

- 순차 데이터를 MLP와 RNN이 각각 어떻게 처리하는지 대비
- MLP는 입력을 펼쳐 **순서를 잃고**, RNN은 시점을 이어가며 **순서를 기억**
- "지금 값"이 아니라 "변해온 패턴"이 중요하다는 주장의 근거

**MLP** 는 각 샘플을 독립적으로 처리하고 입력을 펼쳐서 다루기 때문에 시간 순서라는 개념이 사라집니다. 반면 **LSTM** 은 시퀀스를 기억해 시간에 따른 변화 패턴을 학습합니다.

```{admonition} 교재 인용 — MLP의 한계
:class: tip

*Deep Learning with Python* 10장: 독일 예나 날씨 데이터로 온도 예측 실험에서 **MLP 모델이 상식적인 기준선**("내일 온도는 오늘과 같다")**조차 이기지 못함**. 하지만 LSTM은 기준선을 넘어서는 성능을 보임.
```

---

### ② LSTM의 직관

LSTM(Long Short-Term Memory)의 핵심은 **무엇을 기억하고 무엇을 버릴지** 스스로 학습한다는 데 있습니다. 일반 RNN은 기울기 소실 때문에 먼 과거의 정보가 현재에 영향을 주지 못하는데, LSTM은 세 개의 게이트로 이를 해결합니다. **입력 게이트** 는 새 정보를 얼마나 받아들일지, **망각 게이트** 는 과거 정보를 얼마나 잊을지, **출력 게이트** 는 지금 무엇을 내보낼지를 조절합니다.

```{figure} ../../lecture/images/s2_2_img18.png
:alt: LSTM 셀 구조와 게이트 메커니즘
:width: 67%

LSTM 셀 구조와 게이트 메커니즘
```

- LSTM 셀 내부의 **입력·망각·출력 3개 게이트** 구조를 도식화
- 게이트가 "무엇을 기억하고 무엇을 버릴지"를 학습 → 장기 의존성 유지
- 일반 RNN의 기울기 소실 문제를 어떻게 우회하는지 설명

```{admonition} 교재 인용 — LSTM의 수명
:class: tip

*Deep Learning with Python* 1장: LSTM은 **1997년에 개발된 이후 거의 변하지 않고 여전히 널리 사용**되고 있음. 그만큼 설계가 훌륭했다는 뜻.
```

**엔진 마모 예시**:

엔진 마모에 적용해 보면, **초반 사이클** 에는 망각 게이트가 이전 정보를 유지하면서 입력 게이트가 새로운 정상 패턴을 조금씩 받아들이고, **마모가 진행** 되면 온도 상승 패턴이 누적되어 LSTM이 그 변화를 기억하며, **말기 사이클** 에는 누적된 마모 패턴을 바탕으로 RUL이 낮다는 신호를 출력합니다.

**슬라이딩 윈도우 시퀀스 구성**:

입력은 슬라이딩 윈도우로 구성합니다. window_size를 30으로 두면 사이클 1~30을 (30×14) 시퀀스로 묶어 LSTM에 넣어 RUL을 예측하고, 한 칸씩 밀어 2~31, 3~32… 식으로 계속 샘플을 만듭니다.

```{figure} ../../lecture/images/s2_2_img20.png
:alt: 슬라이딩 윈도우 입력 구성
:width: 67%

슬라이딩 윈도우 입력 구성
```

- 연속 사이클을 window_size 단위로 잘라 **(시점 × 센서)** 시퀀스를 만드는 과정
- 한 칸씩 밀며 다음 샘플을 생성 → 시간 흐름을 입력에 담음
- LSTM이 받는 입력 텐서 (30, 14)가 어떻게 구성되는지 보여줌

*Keras `timeseries_dataset_from_array` 함수가 이 슬라이딩 윈도우를 자동 생성*

**Keras LSTM 모델**:

```python
import tensorflow as tf

model = tf.keras.Sequential([
    tf.keras.layers.LSTM(64, input_shape=(30, 14),
                         return_sequences=False),
    tf.keras.layers.Dropout(0.2),
    tf.keras.layers.Dense(32, activation='relu'),
    tf.keras.layers.Dense(1)  # 활성화 없음 — 회귀 문제
])
model.compile(optimizer='adam', loss='mse', metrics=['mae'])
```

여기서 `input_shape=(30, 14)` 는 30개 사이클 × 14개 센서값을 뜻하고, `return_sequences=False` 는 마지막 시점의 출력만 반환하라는 의미입니다. 마지막 `Dense(1)` 은 활성화 함수 없이 숫자를 그대로 내보내는 회귀 출력입니다.

### ③ 과적합 방지: Dropout과 Early Stopping

딥러닝에서 가장 큰 적은 **과적합(overfitting)**.

```{admonition} 교재 인용 — 최적화와 일반화의 긴장
:class: tip

*Deep Learning with Python* 5장: **"머신러닝의 근본적인 문제는 최적화와 일반화 사이의 긴장"**
- 최적화: 학습 데이터에 최대한 잘 맞추는 것
- 일반화: 새로운 데이터에도 잘 작동하는 것
- 이 둘이 항상 같은 방향으로 가지 않는 것이 문제
```

**과적합 발생 신호**:

과적합은 학습 손실은 계속 떨어지는데 검증 손실은 어느 순간부터 다시 올라가는 형태로 나타납니다. 두 곡선이 갈라지기 시작하는 그 지점이 최적 모델이 있는 시점입니다.

→ 교재 5장에서 **"정준 과적합 패턴(the canonical overfitting pattern)"** 이라고 부름. 어떤 모델, 어떤 데이터셋에서도 보편적으로 나타나는 패턴.

**Dropout** — 학습 중 뉴런의 일부를 무작위로 비활성화:

```{figure} ../../lecture/images/s2_2_img21.png
:alt: Dropout 동작 원리
:width: 67%

Dropout 동작 원리
```

- 학습 중 일부 뉴런을 **무작위로 꺼서(0으로)** 비활성화하는 모습
- 특정 뉴런 과의존을 막아 일반화 성능을 높임(앙상블 효과)
- 추론 시에는 전부 켜짐 — 학습할 때만 적용됨

Geoffrey Hinton은 은행에서 담당 직원을 무작위로 바꾸면 공모 사기를 막을 수 있다는 데서 Dropout의 아이디어를 얻었습니다. 마찬가지로 뉴런을 무작위로 꺼 버리면 특정 뉴런에 대한 과의존이 사라져 일반화 성능이 좋아집니다. 비율은 보통 0.2~0.3을 사용하며, 매 배치마다 다른 네트워크를 학습하는 셈이라 앙상블과 비슷한 효과를 내고, 추론 시에는 자동으로 비활성화됩니다.

**Early Stopping** — 검증 손실이 더 이상 개선되지 않으면 학습 중단:

```python
early_stopping = tf.keras.callbacks.EarlyStopping(
    monitor='val_loss',
    patience=10,                # 10 epoch 동안 개선 없으면 중단
    restore_best_weights=True   # 최적의 가중치 복원
)

model.fit(
    X_train, y_train,
    validation_split=0.2,       # 학습 데이터의 20%를 검증용으로 분리
    epochs=200,
    callbacks=[early_stopping]
)
```

```{figure} ../../lecture/images/s2_2_img22.png
:alt: 과적합 발생: 학습/검증 손실 곡선
:width: 67%

과적합 발생: 학습/검증 손실 곡선
```

- 학습 손실은 계속 감소하지만 **검증 손실은 어느 시점부터 다시 상승**
- 두 곡선이 갈라지는 지점이 **과적합 시작점**
- Early Stopping이 그 직전의 최적 모델을 복원하는 근거

**표준 워크플로우** (*Deep Learning with Python* 5장):

표준 워크플로우는 이렇습니다. 먼저 **과적합이 일어날 수 있는 모델** 을 만든 다음, Dropout·Early Stopping 같은 정규화 기법으로 과적합을 제어합니다. "완벽한 적합을 찾으려면 먼저 과적합을 해야 한다"는 것이 5장의 핵심 메시지입니다.

### 심화: 표현 학습 (Representation Learning)

딥러닝의 핵심은 데이터를 더 나은 **"표현(representation)"** 으로 변환하는 것. 각 층이 점점 더 추상적인 특징을 학습:

구체적으로 **입력층** 은 원본 센서값을, **은닉층 1** 은 진동 주기나 온도 변화율 같은 지역적 패턴을, **은닉층 2** 는 마모 추세나 이상 징후 같은 전역적 패턴을 학습하고, **출력층** 이 최종 RUL 예측값을 냅니다.

이런 "자동 특징 추출"이 딥러닝을 전통 ML과 구분하는 핵심.

```{figure} ../../lecture/images/s2_2_img32.png
:alt: 딥러닝 표현 학습: 층별 특징 추출
:width: 67%

딥러닝 표현 학습: 층별 특징 추출
```

- 신경망이 층을 거치며 **점점 더 추상적인 특징**을 학습하는 모습
- 입력(원본 센서) → 지역 패턴 → 전역 패턴(마모 추세) → RUL
- "자동 특징 추출"이 딥러닝과 전통 ML을 가르는 지점

```{figure} ../../lecture/images/s2_2_img38.png
:alt: 신경망 학습: 순전파와 역전파
:width: 67%

신경망 학습: 순전파와 역전파
```

- 예측을 만드는 **순전파**와 오차로 가중치를 갱신하는 **역전파**의 흐름
- 신경망이 어떻게 학습되는지의 기본 원리
- 위의 표현 학습이 실제로 일어나는 메커니즘
