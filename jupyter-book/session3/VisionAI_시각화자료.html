핵심을 한눈에 정리한 시각 자료 2종 — ① YOLO 아키텍처 변천사(YOLOv1 → YOLO26), ② 행동 인식의 RGB 기반 vs Skeleton 기반 비교. 관통하는 메시지는 “정확도 경쟁 → 배포 친화성(후처리·속도) 경쟁”으로의 이동입니다.

01

YOLO 아키텍처 변천사

단일 스테이지 검출의 시작(v1)부터 NMS-free 표준화(YOLO26)까지. v10이 후처리(NMS) 제거의 방향을 열었고, v12·v13의 attention 설계를 거쳐 YOLO26에서 native NMS-free가 표준이 됩니다.

기반 CNN (v1–v5) anchor-free (v8) 후처리 혁신 (v9–v11) attention 중심 (v12–v13) NMS-free 표준화 (v26)
2016YOLOv1Joseph Redmon

단일 스테이지 grid 예측의 시작 — 검출을 회귀 문제로 재정의해 한 번의 forward로 박스+클래스를 예측.

2018YOLOv3Redmon

3개 스케일 multi-scale 예측, Darknet-53 backbone — 작은 객체 검출이 크게 개선.

2020YOLOv4 / YOLOv5Bochkovskiy / Ultralytics

v4: Mosaic 증강·CSPDarknet. v5: PyTorch 모듈화와 배포 생태계 — 산업 적용의 사실상 표준을 만듦.

2023YOLOv8UltralyticsNMS

anchor-free + decoupled head + C2f 모듈, 검출·분할·pose·분류를 아우르는 멀티태스크. 단, 추론 시 NMS 후처리에 여전히 의존.

2024YOLOv9 · YOLOv10 · YOLO11Wang / THU-MIG / UltralyticsNMS-FREE

v9: PGI·GELAN. v10(칭화대): NMS-free dual label assignment — 학습은 one-to-many, 추론은 one-to-one. v11(9월): C3k2 backbone·C2PSA 공간 attention.

YOLOv10 — COCO 52.7% AP · YOLOv9-C 대비 latency 46% ↓

2025YOLOv12 · YOLOv13Tian 등 (NeurIPS) / Lei 등ATTENTION

attention-centric 설계로 전환 — Area Attention(A2), R-ELAN, FlashAttention. 정확도는 올랐으나 NMS·DFL을 유지해 latency·export 부담이 남음.

YOLOv12-N — 40.6% mAP @ 1.64 ms (T4) · v10-N/v11-N 대비 +2.1%/+1.2%

2026YOLO26UltralyticsNMS-FREEEDGE

native end-to-end NMS-free + DFL 제거로 export 단순화. ProgLoss·STAL(소형 객체)·MuSGD 옵티마이저. 엣지·로봇·모바일에 최적화.

CPU ONNX 추론 43% ↑ (vs YOLO11n) · 2025.9 YOLO Vision London 공개 → 2026.1 릴리스

⚠ 실무 주의: 최신 버전 = 항상 프로덕션 최적은 아니다. YOLOv12는 attention 블록 때문에 학습 불안정·메모리 증가·CPU 처리량 저하가 있어, Ultralytics는 프로덕션에 YOLO11 또는 YOLO26을 권장한다.
02

Action Recognition — RGB vs Skeleton

작업자 위험 행동 인식의 두 접근. Skeleton 표현 자체가 강력한 전처리/특징공학 선택이라는 점이 핵심 — 픽셀을 버리고 관절만 남겨 배경·조명에 robust하고 가볍게 만든다.

1 RGB 기반

픽셀(프레임/클립)을 직접 사용

입력 영상 스트림CCTV / 현장 카메라
프레임 샘플링 · 클립 구성정규화 / 데이터 증강
3D-CNN (I3D · C3D) / Two-Stream시공간 특징 추출
행동 분류 (위험 / 정상)

장점

  • 외형·도구·PPE 등 맥락 정보 풍부
  • 객체 상호작용·환경 단서 보존
  • pose 추정 의존성 없음

단점

  • 배경·조명 변화에 취약
  • 3D conv로 연산이 무거움
  • 대량의 영상 라벨 필요

  • 도구·복장(PPE) 판단 어려움
  • 미세 동작 정보 손실 가능
