# 생성 AI 합성 데이터 -- 5년간 Diffusion 기반 결함 생성의 발전 흐름

```{admonition} 이 섹션의 목적
:class: note

본격적인 이론(Diffusion, 프롬프트 생성, 합성 데이터 검증)에 들어가기 전, **"왜 지금 이 기술인가, 그리고 어디로 가고 있는가"** 의 큰 그림을 먼저 잡습니다. 세부 원리는 4-1 이후에서 다룹니다.
```

## 0-1. 문제 제기

Session 1에서 본 것처럼 제조 데이터는 **정상은 넘치고 결함은 희소한** 극단적 불균형 구조입니다. Session 1은 SMOTE로, Session 3은 임계값 조정으로 대응했지만, 이미지에서는 픽셀 보간이 통하지 않습니다(스크래치와 버블을 평균 내면 의미 없는 얼룩이 됩니다).

그래서 Session 4의 접근은 **데이터를 생성해서 부족분을 채우는 것**입니다. 다만 핵심은 "그럴듯한 그림"을 만드는 게 아니라, **결함 위치 라벨(마스크)까지 정렬된 학습용 데이터**를 만들어 **탐지기 성능을 실제로 올리는 것**입니다.

```{admonition} 한 줄 정의
:class: important

합성 결함 생성의 목표 = "예쁜 결함 이미지"가 아니라 **"라벨까지 정렬된, 믿을 수 있는 학습 데이터"**.
```

그런데 이 기술은 지난 5년간 빠르게 모습을 바꿔 왔습니다. 어떤 방향으로 진화했는지를 먼저 훑으면, 4-1~4-3의 세부 기법이 전체 지도 위 어디에 있는지가 분명해집니다.

---

## 0-2. 이론

### STEP 01 -- 5년의 진화: 더 그럴듯한 그림에서 라벨까지 정렬된 데이터로

Diffusion 기반 결함 생성은 2021년 이후 대략 여섯 단계를 거쳐 왔습니다.

```{mermaid}
flowchart LR
    P0["Phase 0 (2020-21)<br/>토대<br/>DDPM"] --> P1["Phase 1 (2022)<br/>Latent+Text<br/>Stable Diffusion"]
    P1 --> P2["Phase 2 (2023)<br/>제어<br/>ControlNet·LoRA"]
    P2 --> P3["Phase 3 (2024)<br/>목적 특화<br/>이미지-마스크 쌍"]
    P3 --> P4["Phase 4 (2025)<br/>Training-free<br/>프롬프트 기반"]
    P4 --> P5["Phase 5 (2026)<br/>Agentic·VLM<br/>일반화"]
```

- **Phase 0 (2020–21) 토대** — DDPM으로 diffusion이 고품질 생성기로 자리 잡음. 아직 결함 생성엔 GAN이 주류.
- **Phase 1 (2022) Latent + Text** — Stable Diffusion이 잠재 공간으로 비용을 낮추고 텍스트 조건부 생성을 엶. Textual Inversion·DreamBooth 등장.
- **Phase 2 (2023) 제어** — ControlNet으로 "여기에, 이 결함을"이라는 공간 제어가 가능해짐. LoRA로 소수 샘플 적응이 실용화.
- **Phase 3 (2024) 목적 특화** — AnomalyDiffusion·DualAnoDiff 등이 외형(텍스트)과 위치(마스크)를 분리 학습해 **이미지-마스크 쌍**을 생성. 단순 증강과 갈리는 지점.
- **Phase 4 (2025) Training-free** — 생성기를 제품마다 다시 학습하지 않고, 사전학습 모델 + 프롬프트만으로 결함 생성. 신제품에도 즉시 적용.
- **Phase 5 (2026) Agentic·VLM** — 크로스모달·에이전트·일반주의 모델로 확장.

### STEP 02 -- 변화를 관통하는 5개 축

같은 진화를 다섯 차원으로 압축하면, 모든 축이 한 방향(왼쪽 → 오른쪽)으로 움직였습니다.

| 축 | 변화 방향 |
|------|----------|
| **조건 (Conditioning)** | 무조건 → 클래스 → 텍스트 → 텍스트＋마스크 → 크로스모달·에이전트 |
| **출력 (Output)** | 이미지 한 장 → 이미지 ＋ 정렬된 마스크(라벨) |
| **학습 비용 (Cost)** | 전체 학습 → 미세조정 → few-shot 적응 → training-free / zero-shot |
| **연산 공간 (Space)** | 픽셀 공간 → 잠재(latent) 공간 |
| **평가 목표 (Goal)** | 사실성 → 제어·다양성 → 마스크 정렬도 → 미지 제품 일반화 |

```{admonition} 핵심 직관
:class: important

diffusion 결함 합성은 *"더 그럴듯한 이미지를 만드는 일"* 에서 *"원하는 결함을 지정해, 라벨까지 정렬된 채로, 학습 없이 만들어내는 일"* 로 바뀌어 왔습니다.
```

### STEP 03 -- "학습 없이 탐지하니 생성은 필요 없다?" — 흔한 오해

최근 흐름이 training-free / zero-shot으로 간다는 말을 들으면 "그럼 합성은 이제 필요 없냐"는 의문이 생깁니다. 결론은 **그렇지 않습니다.** "training-free"는 두 군데에 쓰이는데, **생성** 쪽에서는 "생성기를 제품마다 다시 학습 안 해도 된다"(즉 합성이 더 싸진 것)는 뜻이지 합성을 안 한다는 뜻이 아니고, **탐지** 쪽의 zero-shot은 별개 트랙입니다.

```{admonition} 합성 데이터가 특히 유효한 영역
:class: tip

- **픽셀 단위 정밀 탐지·세그멘테이션** — zero-shot은 "결함 유무"는 맞혀도 "정확히 어느 픽셀"엔 약함. 면적·형태를 재야 하면 마스크 달린 합성 데이터가 필요 (예: 강판 스크래치 면적 측정).
- **고신뢰 양산 QC** — 놓치면 비싼 환경에선 (실제＋합성)으로 학습한 최고 성능 탐지기가 유리 (예: 반도체 웨이퍼).
- **희소·신규 결함 보강** — 거의 안 나오는 유형을 의도적으로 충분히 생성해 학습 분포의 구멍을 메움.
- **라벨링 비용 절감** — 생성 단계에서 마스크가 함께 나오므로 수작업 라벨링을 줄임.
```

이 "잘 만들면 도움, 잘못 만들면 독"이라는 균형은 4-3(합성 데이터 고도화)에서 실험으로 검증합니다.

### STEP 04 -- 또 다른 길: 생성 vs VLM·파운데이션 모델

합성만이 답은 아닙니다. CVPR 2025의 **VAND 3.0 챌린지**는 시각 이상탐지를 두 트랙으로 겨뤘는데, 서로 다른 두 철학을 잘 보여줍니다.

- **Category 1 (Adapt & Detect)** — MVTec AD 2 데이터셋, **정상 이미지만으로 학습하는 one-class** 탐지(픽셀 SegF1 평가). 합성은 이 탐지기를 보강하는 레버로 쓰임. 우승: ISVL, RoBiS.
- **Category 2 (VLM Anomaly)** — MVTec LOCO 데이터셋, **비전-언어 모델(VLM) + few-shot**으로 구조적·**논리적** 이상까지 탐지. 합성 없이 거대 모델의 사전지식을 활용. 우승: FastLogSAD, UniVAD++.

```{admonition} 두 갈래는 경쟁이 아니라 보완
:class: note

- **합성(generation)** = 데이터를 만들어 탐지기를 강하게 하는 길 → 픽셀 정밀·양산 고정 제품에 강점
- **VLM·파운데이션 모델** = 거대한 사전지식을 빌려 쓰는 길 → 라벨 없는 신제품·논리적 이상에 강점
- 상황에 따라 고르거나 결합합니다.
```
