# 심화 ① · ② — 결함 이미지를 "만들어내는" 두 가지 방법

```{admonition} 이 문서의 위치
:class: note

앞선 sec0〈생성 AI 합성 데이터〉의 진화 타임라인에서 본 흐름 중, **Phase 1~4의 핵심 도구 두 가지**를 자세히 살펴봅니다. 하나는 *모델에게 우리 결함을 가르치는 법*(파인튜닝), 다른 하나는 *어떤 결함을 만들지 말로 지시하는 법*(텍스트 생성)입니다. 둘은 경쟁이 아니라 **같이 쓰는 두 개의 레버**입니다.
```

## 들어가기 전에: 왜 "생성"이 필요한가 (1분 요약)

제조 라인에서 결함은 아주 드물게 나옵니다. 정상 제품 사진은 수만 장인데 결함 사진은 몇 장~수십 장뿐인 경우가 흔하죠. 그런데 결함을 잘 잡는 AI(탐지기)를 학습시키려면 **결함 사례가 충분히** 있어야 합니다. 이 부족분을 진짜 결함 대신 **인공적으로 만들어 채우는 것**이 결함 이미지 생성입니다. 핵심 목표는 단순히 그림을 만드는 게 아니라, **"결함이 어디 있는지 표시(마스크)까지 붙은 학습용 데이터"** 를 얻는 것입니다.

### 그런데 — "요즘은 학습 없이도 탐지한다던데, 생성이 아직도 필요해?"

최근 흐름이 **training-free / zero-shot**(학습 샘플 없이 미지의 제품도 바로 탐지)으로 간다는 얘기를 들으면, "그럼 합성 데이터를 만드는 단계는 이제 필요 없는 거 아니냐"는 의문이 자연스럽게 생깁니다. 결론부터 말하면 **그렇지 않습니다.** 여기엔 흔한 오해가 하나 섞여 있습니다.

"training-free"라는 말은 두 군데에 쓰입니다. **생성** 쪽에서 쓰면 "그림 만드는 모델을 제품마다 다시 학습시키지 않아도 된다"는 뜻이지, **합성 데이터를 안 만든다는 뜻이 아닙니다.** 즉 생성 단계는 사라진 게 아니라 **더 싸지고 쉬워진** 것입니다. 한편 **탐지** 쪽에서 쓰는 zero-shot(예: CLIP 기반)은 학습 없이 이상을 잡는 별개의 트랙인데, 이것조차 합성을 완전히 대체하지는 못합니다.

```{admonition} 비유
:class: note

"조립 라인을 자동화했다"가 "공장이 더 이상 부품을 안 만든다"를 뜻하진 않습니다. 만드는 게 **싸지고 빨라졌을 뿐**, 부품은 여전히 필요합니다.
```

### 합성 데이터가 *특히* 유효한 영역 (예시와 함께)

- **픽셀 단위 정밀 탐지·세그멘테이션** — zero-shot 방식은 "이 이미지에 결함이 있다"(이미지 단위)는 곧잘 맞히지만, "정확히 **어느 픽셀**이 결함인가"는 약합니다. 결함의 **면적·형태를 정밀하게 재서 품질 등급을 매겨야 할 때**는, 마스크가 달린 데이터로 학습한 세그멘테이션 모델이 필요하고 — 그 마스크를 정렬해서 함께 찍어내는 게 바로 생성의 고유 강점입니다.
  *예: 강판 표면 스크래치의 길이·면적 측정, PCB 납땜 불량 위치를 픽셀 단위로 표시.*

- **놓치면 비싼 양산 QC(고신뢰 요구)** — zero-shot은 아직 잘 학습된 모델(supervised)의 정밀도를 따라잡지 못합니다. 불량을 한 번 놓치는 비용이 큰 환경에서는 **(실제 + 합성) 데이터로 학습한 최고 성능 탐지기**가 정답에 가깝습니다.
  *예: 자동차 안전 부품, 반도체 웨이퍼 검사.*

- **희소·신규 결함 보강** — 1년에 몇 번 나올까 말까 한 특정 결함은 데이터가 너무 적어 학습이 안 됩니다. 이런 유형을 **의도적으로 충분히 생성**해 학습 분포의 구멍을 메웁니다.
  *예: 드물게 발생하는 주조품 내부 기공(porosity)을 골라 대량 생성.*

- **라벨링 비용 절감** — 사람이 결함 위치를 픽셀 단위로 일일이 칠하는 작업은 매우 비쌉니다. 생성은 **이미지와 마스크(정답)를 동시에** 내놓으므로 이 수작업을 크게 줄여 줍니다.

- **통제된 평가셋 제작** — "이런 결함을 이만큼" 만들어, 탐지기를 일관된 기준으로 체계적으로 시험할 수 있습니다.

### 한 줄 정리: 경쟁이 아니라 역할 분담

라벨이 전혀 없는 **신제품에 즉시 대응**해야 할 땐 zero-shot 탐지가, **양산 라인의 고정된 제품에서 최고 정밀도**가 필요할 땐 합성 증강 + supervised가 강점입니다. 그리고 전제는 항상 하나 — **"잘" 생성해야** 한다는 것. 진짜 결함과 동떨어진 합성물은 오히려 탐지기를 망치므로, 뒤에서 볼 "믿을 수 있게 생성하기"가 이 분야의 핵심 과제로 남아 있습니다.

## ① Diffusion 모델 파인튜닝 — 제조 결함 이미지 생성

### 파인튜닝이란, 그리고 왜 필요한가

**Diffusion 모델**(예: Stable Diffusion)은 인터넷의 수십억 장 이미지로 학습된 **범용 그림 생성기**입니다. 고양이, 풍경, 사람은 기가 막히게 그립니다. 그런데 "우리 공장 PCB 기판의 미세 납땜 균열" 같은 건 학습 중에 거의 본 적이 없습니다. 그래서 그냥 "PCB 결함 그려줘" 하면 엉뚱하거나 비현실적인 결과가 나옵니다.

**파인튜닝(fine-tuning)** 은 이 범용 모델을 우리가 가진 **소수의 실제 결함 사진으로 "추가 교육"** 시켜, 우리 도메인에 맞게 미세 조정하는 것입니다.

```{admonition} 비유
:class: note

외국어를 유창하게 하는 통역사(범용 모델)에게, 우리 회사 전문용어 사전(소수 결함 샘플)을 며칠 가르치는 것. 언어를 처음부터 가르치는 게 아니라, 이미 잘하는 사람을 우리 업무에 맞게 길들이는 것입니다.
```

### 제조 현실의 함정: "몇 장뿐"이라는 문제 (few-shot)

결함 사진이 몇 장밖에 없는 상태에서 모델 전체를 다시 학습시키면 두 가지 사고가 납니다.

- **과적합(overfitting)** — 그 몇 장을 통째로 외워버려, 거의 똑같은 그림만 복사합니다. 다양성이 없어 학습 데이터로서 가치가 떨어집니다.
- **망각(catastrophic forgetting)** — 원래 알던 일반적인 그림 지식을 잊어버려, 오히려 품질이 망가집니다.

```{admonition} 비유
:class: note

통역사에게 우리 용어 10개만 며칠 내내 외우게 하면, 그 10개는 달달 외우지만(과적합) 정작 평소 하던 통역 실력이 녹슬어 버리는(망각) 것과 같습니다.
```

그래서 "**조금만, 효율적으로**" 손대는 기법들이 발전했습니다. 이를 **파라미터 효율적 파인튜닝(PEFT)** 이라고 부릅니다.

### 대표 기법 세 가지

| 기법 | 한 줄 개념 | 비유 | 특징 |
| --- | --- | --- | --- |
| **Textual Inversion** | 모델은 안 건드리고, "이 결함"을 가리키는 **새 단어(임베딩) 하나**만 학습 | 모델 머릿속 사전에 **신조어 한 개** 등록 | 가장 가볍다. 대신 표현할 수 있는 범위가 제한적 |
| **DreamBooth** | 소수 샘플로 모델을 **살짝 재학습**하되, 원래 지식을 잊지 않도록 안전장치를 둠 | 특정 결함을 **집중 과외**시키되 일반 상식은 유지 | 특정 대상을 충실히 재현. 학습이 비교적 무거움 |
| **LoRA** | 거대한 본체 가중치는 **얼리고(고정)**, 작은 보조 모듈만 추가로 학습 | 본체는 그대로, 작은 **확장 모듈**만 끼웠다 뺐다 | 학습량이 수백~수천 배 적어 빠르고 가볍다. 현재 가장 실용적 |

세 가지는 종종 **함께** 쓰입니다. 예를 들어 "LoRA로 가볍게 적응시키고, 위치 제어는 별도 신호로 준다"는 식입니다.

### 가장 중요한 산출물: 이미지 + 마스크 "쌍"

여기가 단순 데이터 증강과 결정적으로 갈리는 지점입니다. 결함 **그림만** 만들면 반쪽짜리입니다. 탐지기를 제대로 학습시키려면 **"결함이 정확히 어느 픽셀에 있는지" 표시한 마스크**가 함께 있어야 합니다(이 마스크가 곧 정답 라벨입니다).

최신 방법(예: **AnomalyDiffusion**)은 결함의 **외형(무엇)** 과 **위치(어디)** 를 분리해서 학습합니다. 외형은 "스크래치/오염 같은 종류"를 나타내는 학습된 단어로, 위치는 마스크로 따로 다룹니다. 이렇게 분리하면 **원하는 위치에 원하는 종류의 결함을 찍어내면서, 정렬된 마스크까지 한 번에** 대량 생산할 수 있습니다.

### "잘" 파인튜닝했는지 판단하는 기준

생성된 결함이 (1) **진짜 같고**, (2) **다양하고**, (3) **마스크와 정확히 맞아야** 합니다. 그리고 최종 시험은 단 하나 — **이 합성 데이터로 학습시킨 탐지기의 성능이 실제로 올라가는가** 입니다.

### 잠깐 — "내 강아지 사진으로 모델 길들이기"와 뭐가 다를까?

방금 본 파인튜닝 기법(LoRA, DreamBooth 등)은 사실 "내 강아지 사진 몇 장으로 생성 모델을 길들이기" 같은 **일반적인 용도와 도구가 똑같습니다.** 출발점(범용 모델), 사용하는 기법, 소수 샘플로 학습한다는 점, 과적합·망각을 조심한다는 점까지 공유하죠. 그래서 **메커니즘은 거의 같지만**, 목적과 산출물·평가 기준이 본질적으로 다릅니다.

| 관점 | 범용 이미지 파인튜닝 (예: 내 강아지·특정 화풍) | 제조 이상탐지용 파인튜닝 |
| --- | --- | --- |
| **최종 목적** | 이미지 생성 그 자체 | 생성은 수단 — **결함 탐지기 성능 향상**이 목적 |
| **"잘했다"의 기준** | 보기 좋고, 원하는 대상을 닮았는가 | 이 데이터로 학습한 **탐지기 성능이 실제로 오르는가** |
| **핵심 산출물** | 이미지 | 이미지 **＋ 정렬된 마스크(라벨)** 쌍 |
| **학습 대상의 성격** | 전체 객체·주체·스타일 (전체적) | 정상 위의 **작은 결함** (국소적, 배경과 자연스럽게 섞여야) |
| **"충실도"의 뜻** | 대상이 닮았는가 | **실제 라인에서 나올 법한 결함인가** |
| **데이터 상황** | 비슷한 이미지가 인터넷에 흔함 | 정상 多·결함 極少, 모델이 거의 본 적 없는 도메인 |
| **위치 제어** | 별로 중요하지 않음 | **위치 = 라벨**이라 마스크로 정밀 지정 필수 |

```{admonition} 비유
:class: note

같은 카메라(파인튜닝 기법)라도, **인물 사진**을 찍을 때와 **의료 X선**을 찍을 때 "잘 찍었다"의 기준은 완전히 다릅니다. 인물 사진은 예쁘면 되지만, X선은 "진단에 실제로 도움이 되는가"가 전부죠. 제조 결함 파인튜닝은 후자에 가깝습니다 — 예쁜 그림이 아니라, **라벨까지 맞은 현실적인 결함으로 탐지기를 더 잘 만드는 것**이 목적입니다.
```

## ② Defect Generation — 텍스트 기반 결함 생성

### 핵심 아이디어

①번 파인튜닝이 "모델에게 **우리 결함을 가르치는 것**"이라면, 텍스트 기반 생성은 "**어떤 결함을 만들지 말로 지시하는 것**"입니다.

예를 들어 이렇게 주문합니다.
- "금속 표면에 **가느다란 수직 스크래치**"
- "캡슐 표면의 **갈색 오염 얼룩**"
- "PCB 기판 모서리의 **납땜 누락**"

그러면 그 설명에 맞는 결함 이미지가 나옵니다.

### 글이 어떻게 그림을 바꾸는가 (부담 없이)

모델 안에는 **글과 그림을 같은 좌표 공간에 놓고 비교하는 번역기**가 들어 있습니다(이걸 **CLIP**이라고 부릅니다). "스크래치"라는 단어를 숫자 묶음(벡터)으로 바꾸고, 그림을 그려가는 과정에서 모델이 **그 단어 벡터를 계속 곁눈질(attention)** 하며 픽셀을 채웁니다.

```{admonition} 비유
:class: note

화가에게 **주문서**를 건네는 것. 주문서(텍스트)가 구체적일수록 결과가 원하는 것에 가까워집니다. "스크래치"보다 "길이 2cm의 가느다란 수직 스크래치"가 더 정확한 그림을 끌어냅니다.
```

### 가장 중요한 한계: 텍스트는 '무엇'엔 강하고 '어디'엔 약하다

텍스트만으로는 **"스크래치를 만들어라"** 까지는 잘 되지만, **"정확히 이 위치에"** 는 잘 안 됩니다. 그래서 실무 방법은 거의 항상 **텍스트(무엇) + 마스크(어디)** 를 함께 줍니다.

```{admonition} 비유
:class: note

주문서에 "스크래치"라고 적고, 함께 건넨 **도면에 동그라미로 "여기"** 를 표시해 주는 것. 그래야 화가가 원하는 자리에 정확히 그립니다.
```

이게 인트로 타임라인에서 본 "텍스트＋마스크" 단계(Phase 3)이고, 이 덕분에 **이미지와 정렬된 마스크(라벨)를 함께** 얻을 수 있습니다.

### 두 갈래 접근

**(가) 학습이 필요한 방식** — ①번 파인튜닝과 결합합니다. 우리 제품으로 적응시킨 모델에 텍스트/마스크를 줘서 생성합니다. 충실도(진짜 같음)가 높지만, 제품마다 학습 비용이 듭니다.

**(나) 학습이 필요 없는 방식 (training-free / zero-shot)** — 범용 모델을 **그대로 둔 채** 텍스트 프롬프트와 약간의 내부 조작(어텐션 조정)만으로 결함을 만듭니다. 빠르고 신제품에도 즉시 적용되지만, 엉뚱한 결과가 섞일 수 있어 **생성물을 자동으로 걸러내는 검증 장치(필터링)** 가 중요합니다.

### 왜 "잘 생성하기"가 어렵고 또 중요한가

대충 만든 합성 결함은 오히려 독이 됩니다. 흔한 실패는 세 가지입니다.

- **경계가 부자연스러움** — 결함과 배경 경계에 "오려 붙인 티"가 남음
- **텍스트와 다른 걸 만듦** — "스크래치"라고 했는데 엉뚱한 무늬가 나옴
- **진짜 결함과 동떨어짐** — 현실에 없는 형태라 탐지기를 헷갈리게 함

그래서 연구는 **경계의 자연스러움, 텍스트-이미지 일치, 마스크 정렬, 생성물 자동 필터링**을 끌어올리는 데 집중합니다. "생성한다"가 아니라 "**믿을 수 있게** 생성한다"가 진짜 과제인 셈입니다.

### 이게 실무에서 주는 가치

- **희소 결함 보강** — 거의 안 나타나는 특정 결함을 의도적으로 충분히 만들어 학습 분포의 구멍을 메움
- **라벨 달린 데이터 확보** — 마스크가 함께 나오므로, 사람이 일일이 결함 위치를 칠하는 비싼 라벨링을 줄임
- **통제된 테스트셋** — "이런 결함을 이만큼" 만들어 탐지기를 체계적으로 평가
