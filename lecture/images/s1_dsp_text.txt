[19] (Normal) Spectral Decomposition
[20] (Normal) The most important topic in this book is spectral decomposition, which is the idea that any signal can be expressed as the sum of sinusoids with different frequencies.
[21] (Normal) The most important mathematical idea in this book is the Discrete Fourier Transform (DFT), which takes a signal and produces its spectrum. The spectrum is the set of sinusoids that add up to produce the signal.
[22] (Normal) And the most important algorithm in this book is the Fast Fourier Transform (FFT), which is an efficient way to compute the DFT.
[23] (Normal) For example, Figure 1-3 shows the spectrum of the violin recording in Figure 1-2. The x-axis is the range of frequencies that make up the signal. The y-axis shows the strength or amplitude of each frequency component.

[176] (Normal) Before we go on, let’s see how Chirp is implemented. Here is the class definition:
[177] (Normal) class Chirp(Signal):
[179] (Normal) def __init__(self, start=440, end=880, amp=1.0):
[180] (Normal) self.start = start
[181] (Normal) self.end = end
[182] (Normal) self.amp = amp
[183] (Normal) start and end are the frequencies, in Hz, at the start and end of the chirp. amp is amplitude.
[184] (Normal) Here is the function that evaluates the signal:
[185] (Normal) def evaluate(self, ts):
[186] (Normal) freqs = np.linspace(self.start, self.end, len(ts)-1)
[187] (Normal) return self._evaluate(ts, freqs)
[188] (Normal) ts is the sequence of points in time where the signal should be evaluated; to keep this function simple, I assume they are equally spaced.
[189] (Normal) [IMG]If the length of ts is n, you can think of it as a sequence of  intervals of time. To compute the frequency during each interval, I use np.linspace, which returns a NumPy array of  values between start and end.
[190] (Normal) _evaluate is a private method that does the rest of the math1:
[191] (Normal) def _evaluate(self, ts, freqs):
[192] (Normal) dts = np.diff(ts)
[193] (Normal) dphis = PI2 * freqs * dts
[194] (Normal) phases = np.cumsum(dphis)
[195] (Normal) phases = np.insert(phases, 0, 0)
[196] (Normal) ys = self.amp * np.cos(phases)
[197] (Normal) return ys
[198] (Normal) np.diff computes the difference between adjacent elements of ts, returning the length of each interval in seconds. If the elements of ts are equally spaced, the dts values are all the same.

[215] (Normal) Exponential Chirp
[216] (Normal) When you listen to this chirp, you might notice that the pitch rises quickly at first and then slows down. The chirp spans two octaves, but it only takes 2/3 s to span the first octave, and twice as long to span the second.
[217] (Normal) The reason is that our perception of pitch depends on the logarithm of frequency. As a result, the interval we hear between two notes depends on the ratio of their frequencies, not the difference. “Interval” is the musical term for the perceived difference between two pitches.
[218] (Normal) For example, an octave is an interval where the ratio of two pitches is 2. So the interval from 220 to 440 Hz is one octave and the interval from 440 to 880 Hz is also one octave. The difference in frequency is bigger, but the ratio is the same.
[219] (Normal) As a result, if frequency increases linearly, as in a linear chirp, the perceived pitch increases logarithmically.
[220] (Normal) If you want the perceived pitch to increase linearly, the frequency has to increase exponentially. A signal with that shape is called an exponential chirp.
[221] (Normal) Here’s the ExpoChirp class definition:
[222] (Normal) class ExpoChirp(Chirp):
[224] (Normal) def evaluate(self, ts):
[225] (Normal) start, end = np.log10(self.start), np.log10(self.end)
[226] (Normal) freqs = np.logspace(start, end, len(ts)-1)
[227] (Normal) return self._evaluate(ts, freqs)
[228] (Normal) Instead of np.linspace, this version of evaluate uses np.logspace, which creates a series of frequencies whose logarithms are equally spaced, which means that they increase exponentially.
[229] (Normal) That’s it; everything else is the same as Chirp. Here’s the code that makes one:
[230] (Normal) signal = thinkdsp.ExpoChirp(start=220, end=880)
[231] (Normal) wave = signal.make_wave(duration=1)
[232] (Normal) You can listen to the examples in chap03.ipynb and compare the linear and exponential chirps.

[243] (Normal) Spectrogram
[244] (Normal) To recover the relationship between frequency and time, we can break the chirp into segments and plot the spectrum of each segment. The result is called a Short-Time Fourier Transform (STFT).
[245] (Normal) There are several ways to visualize an STFT, but the most common is a spectrogram, which shows time on the x-axis and frequency on the y-axis. Each column in the spectrogram shows the spectrum of a short segment, using color or grayscale to represent amplitude.

[248] (Normal) wave = signal.make_wave(duration=1, framerate=11025)

[251] (Normal) spectrogram.plot(high=700)
[252] (Normal) seg_length is the number of samples in each segment. I chose 512 because the FFT is most efficient when the number of samples is a power of 2.

[266] (Normal) Leakage
[267] (Normal) In order to explain how make_spectrogram works, I have to explain windowing; and in order to explain windowing, I have to show you the problem it is meant to address, which is leakage.
[268] (Normal) The Discrete Fourier Transform (DFT), which we use to compute Spectrums, treats waves as if they are periodic; that is, it assumes that the finite segment it operates on is a complete period from an infinite signal that repeats over all time. [283] (Normal) Windowing
[284] (Normal) We can reduce leakage by smoothing out the discontinuity between the beginning and end of the segment, and one way to do that is windowing.
[285] (Normal) A “window” is a function designed to transform a non-periodic segment into something that can pass for periodic. Figure 3-5 (top) shows a segment where the end does not connect smoothly to the beginning.
[286] (Normal) Figure 3-5 (middle) shows a “Hamming window”, one of the more common window functions. No window function is perfect, but some can be shown to be optimal for different applications, and Hamming is a good, all-purpose window.
[287] (Normal) Figure 3-5 (bottom) shows the result of multiplying the window by the original signal. Where the window is close to 1, the signal is unchanged. Where the window is close to 0, the signal is attenuated. Because the window tapers at both ends, the end of the segment connects smoothly to the beginning.

[289] (Normal) Figure 3-5. Segment of a sinusoid (top), Hamming window (middle), and product of the segment and the window (bottom).

[299] (Normal) Now that we understand windowing, we can understand the implementation of make_spectrogram. Here is the Wave method that computes spectrograms:
[300] (Normal) #class Wave:
[301] (Normal) def make_spectrogram(self, seg_length):
[302] (Normal) window = np.hamming(seg_length)
[303] (Normal) i, j = 0, seg_length
[304] (Normal) step = seg_length / 2
[306] (Normal) spec_map = {}
[308] (Normal) while j < len(self.ys):
[309] (Normal) segment = self.slice(i, j)
[310] (Normal) segment.window(window)
[312] (Normal) t = (segment.start + segment.end) / 2
[313] (Normal) spec_map[t] = segment.make_spectrum()
[315] (Normal) i += step
[316] (Normal) j += step

[319] (Normal) This is the longest function in the book, so if you can handle this, you can handle anything.
[320] (Normal) The parameter, self, is a Wave object. seg_length is the number of samples in each segment.
[321] (Normal) window is a Hamming window with the same length as the segments.
[322] (Normal) i and j are the slice indices that select segments from the wave. step is the offset between segments. Since step is half of seg_length, the segments overlap by half. Figure 3-6 shows what these overlapping windows look like.
[323] (Normal) [IMG]
[324] (Normal) Figure 3-6. Overlapping Hamming windows.
[325] (Normal) spec_map is a dictionary that maps from a timestamp to a Spectrum.
[326] (Normal) Inside the while loop, we select a slice from the wave and apply the window; then we construct a Spectrum object and add it to spec_map. The nominal time of each segment, t, is the midpoint.
[327] (Normal) Then we advance i and j, and continue as long as j doesn’t go past the end of the Wave.

[497] (Normal) Gaussian Noise
[498] (Normal) We started with uncorrelated uniform (UU) noise and showed that, because its spectrum has equal power at all frequencies, on average, UU noise is white.
