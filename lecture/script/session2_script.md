## 1-1. 문제 제기 (3분)

Session 1에서 Isolation Forest로 이상 탐지를 해봤습니다. 그런데 실제 제조 현장에서는 한 가지 큰 문제가 있습니다.

공장에서는 수만 개의 정상 제품이 생산되는 동안 결함은 소수에 불과합니다. 그리고 이 결함들에는 **라벨이 없습니다**. 이게 뭔지, 저게 뭔지 알 수 없는 거죠.

지도 학습을 쓰려면 충분한 결함 샘플이 있어야 하고, 라벨링도 해야 하고, 새로운 결함 유형이 나타나면 또 다시 학습해야 합니다. 현실적으로 어렵죠.

그래서 핵심 질문은 이겁니다. **라벨 없이, 정상 데이터만으로 이상을 탐지할 수 있는가?**

핵심 아이디어는 간단합니다. **정상만 학습한 모델은 이상을 재현하지 못한다.** 이 차이를 이용하는 거죠.

## 1-2. 이론 (13분)

### ① 비지도 학습의 프레임워크 (3분)

*Hands-On Unsupervised Learning* 1장과 4장 내용입니다.

비지도 학습은 라벨 없이 데이터 자체의 구조를 학습합니다. 이상 탐지에서의 전략은 "정상 데이터만으로 정상이 어떤 모습인지 학습하는 것"입니다.

Session 1에서 배운 PCA, 차원 축소도 비지도 학습의 일종입니다. 고차원 데이터를 저차원으로 압축하면서 중요한 패턴만 남기는 거죠. 하지만 PCA는 선형 변환이라 복잡한 패턴을 잡기 어렵습니다.

오늘 배울 Autoencoder는 비선형 차원 축소를 수행합니다. 더 복잡한 패턴을 학습할 수 있죠.

### ② Autoencoder 구조와 재구성 오차 (5분)

*Deep Learning with Python* 2판 내용입니다.

Autoencoder는 두 부분으로 구성됩니다.

**인코더**는 입력을 압축합니다. 64차원 입력을 32차원으로, 다시 8차원으로 줄입니다. 이걸 **잠재 공간, latent space**라고 부릅니다.

**디코더**는 압축된 표현을 다시 복원합니다. 8차원을 32차원으로, 다시 64차원으로 늘립니다.

손실 함수는 MSE, 평균 제곱 오차입니다. 원본 입력과 복원된 출력이 얼마나 다른가를 측정하죠.

이상 탐지 원리는 이렇습니다. 정상 입력은 인코더가 잘 압축하고 디코더가 잘 복원합니다. 그래서 재구성 오차가 낮습니다. 하지만 이상 입력은 학습한 적 없는 패턴이니까 인코더가 제대로 압축하지 못하고, 디코더도 제대로 복원하지 못합니다. 재구성 오차가 높게 나오는 거죠.

**재구성 오차 = 이상 점수**입니다.

PyTorch로 구현하면 이렇습니다. 인코더는 Linear 64에서 32, ReLU, Linear 32에서 8. 디코더는 Linear 8에서 32, ReLU, Linear 32에서 64. 간단하죠.

재구성 오차는 `(x - x_hat) ** 2`의 `mean(dim=1)`으로 계산합니다.

잠재 공간 차원, latent_dim은 중요한 하이퍼파라미터입니다. 너무 크면 이상도 잘 재구성되어서 이상 탐지가 안 됩니다. 너무 작으면 정상도 재구성하지 못합니다. 적절한 크기를 찾는 게 관건입니다.

### ③ 임계값 설정: 어디서부터 이상인가 (5분)

*Hands-On Unsupervised Learning* 4장 내용입니다.

재구성 오차를 계산했으면, 어디서부터 이상으로 볼 건지 임계값을 정해야 합니다.

세 가지 방법이 있습니다.

첫째, **정상 오차의 95번째 백분위수**. 정상 데이터의 재구성 오차를 정렬해서 상위 5%를 이상으로 간주하는 겁니다.

둘째, **평균 + 3×시그마**. 3-시그마 규칙이라고도 합니다. 정상 오차의 평균에서 표준편차의 3배를 넘으면 이상으로 보는 거죠.

셋째, **도메인 전문가 판단**. 실제 현장에서는 전문가가 경험적으로 임계값을 정하기도 합니다.

트레이드오프가 있습니다. 임계값이 낮으면 민감도는 높아지지만 오탐율도 높아집니다. 안전이 중요한 설비에 적합하죠. 반대로 임계값이 높으면 오탐율은 낮아지지만 이상 탐지를 놓칠 수 있습니다. 품질 검사에 적합합니다.

## 2-1. 문제 제기 (3분)

섹션 2 시작하겠습니다.

이번에는 다른 문제를 다룹니다. "이상인가 아닌가"가 아니라 **"얼마나 더 버틸 수 있는가"**를 예측하는 겁니다. 이걸 **RUL, Remaining Useful Life, 잔여 유효 수명**이라고 부릅니다.

유지보수에는 세 가지 전략이 있습니다.

첫째, **고장 후 수리**. 고장 나면 고치는 거죠. 비용이 최대이고 위험도 최대입니다.

둘째, **예방 정비**. 주기적으로 교체하는 겁니다. 비용은 중간이지만, 아직 쓸 수 있는 부품을 버리게 됩니다.

셋째, **예지 정비**. RUL 예측을 기반으로 교체 시점을 결정하는 겁니다. 비용이 최소이고 위험도 최저입니다.

예를 들어볼까요. 총 수명 300사이클인 엔진이 있습니다. 사이클 1에서 RUL은 299. 사이클 250에서 RUL은 50, 여기서 교체를 결정합니다. 사이클 300에서 RUL은 0, 고장입니다.

핵심 질문: **현재 센서 데이터만으로 RUL을 예측할 수 있는가?**

---

## 2-2. 이론 (13분)

### ① 왜 일반 회귀가 아닌 LSTM인가 (3분)

*ML for Time-Series with Python* 7장 내용입니다.

의문이 드실 겁니다. RUL 예측이면 그냥 선형 회귀나 MLP로 하면 되지 않나?

안 되는 이유는 **순서가 중요하기 때문**입니다.

같은 센서 값이라도 어느 사이클에서 측정됐느냐에 따라 RUL이 완전히 다릅니다. 온도가 300도인데 초반이면 정상일 수 있고, 말기이면 마모 징후일 수 있습니다.

MLP는 입력의 순서를 무시합니다. 그냥 특징들만 펼쳐서 넣죠. 하지만 LSTM은 시퀀스를 기억합니다. 시간에 따른 변화 패턴을 학습하는 거죠.

### ② LSTM의 직관 (5분)

*Deep Learning with Python* 2판 내용입니다.

LSTM, Long Short-Term Memory의 핵심은 **무엇을 기억하고 무엇을 버릴지 학습**한다는 겁니다.

세 가지 게이트가 있습니다.

**입력 게이트**는 새로운 정보를 얼마나 받아들일지 결정합니다.
**망각 게이트**는 기존 정보를 얼마나 잊을지 결정합니다.
**출력 게이트**는 현재 상태에서 무엇을 출력할지 결정합니다.

엔진 마모 예로 생각해 보세요. 초반에는 정상 패턴을 유지합니다. 망각 게이트가 이전 정보를 그대로 유지하면서 입력 게이트는 새로운 정상 패턴을 조금씩 받아들이는 거죠. 마모가 진행되면 온도 상승 패턴이 누적됩니다. LSTM이 이 변화를 기억하는 겁니다. 말기에는 누적된 마모 패턴을 바탕으로 RUL이 낮다는 신호를 출력합니다.

슬라이딩 윈도우를 사용합니다. `window_size=30`이면 사이클 1부터 30까지의 센서 데이터, 30×14 행렬이 LSTM의 입력이 되고, 출력이 RUL 예측값이 됩니다.

Keras로 구현하면: `LSTM(64, input_shape=(30, 14))` → `Dropout(0.2)` → `Dense(32, relu)` → `Dense(1)`. 옵티마이저는 adam, 손실함수는 mse, 메트릭은 mae입니다.

### ③ 과적합 방지: Dropout과 Early Stopping (5분)

딥러닝에서 가장 큰 적은 과적합입니다. 학습 데이터에는 완벽하지만 새로운 데이터에는 못하는 거죠.

**Dropout**은 학습 중에 뉴런의 일부를 무작위로 비활성화하는 기법입니다. 모델이 특정 뉴런에 의존하지 않게 만들어서 일반화 성능을 높입니다. 비율은 보통 0.2~0.3을 씁니다.

**Early Stopping**은 검증 손실이 더 이상 개선되지 않으면 학습을 중단하는 겁니다. `patience=10`이면 10 epoch 동안 검증 손실이 개선되지 않으면 멈춥니다. `restore_best_weights=True`로 최적의 가중치를 복원합니다.

이 두 가지를 조합하면 과적합을 효과적으로 방지할 수 있습니다.

## 3-1. 데이터 확인 + RUL 라벨 설계 (10분)

마지막 섹션입니다. 실제 데이터셋으로 지금까지 배운 걸 종합해서 적용해 보겠습니다.

열 구성은: 유닛 번호, 사이클 번호, 운전 조건 3개, 센서 값 21개입니다.

먼저 Claude에게 EDA를 요청해 보겠습니다. 데이터 형태, 결측치, 유닛 수, 사이클 범위, 센서별 시계열 플롯, 그리고 저분산 센서, 표준편차가 0.01 미만인 센서를 식별합니다.

### RUL 클리핑: 왜 125로 자르는가

중요한 전처리가 하나 있습니다. RUL 클리핑입니다.

엔진 초기, 사이클 1~100 정도의 센서 데이터는 모두 "정상"입니다. RUL이 250이든 300이든 350이든 센서 데이터가 비슷하죠. 이걸 다 학습시키면 모델이 초기 구간에서 불필요하게 노력하게 됩니다.

그래서 RUL 상한을 125로 자릅니다. `RUL = (max_cycle - cycle).clip(upper=125)`. 125 이상은 모두 125로 처리하는 거죠. 이렇게 하면 모델이 의미 있는 예측 범위에 집중할 수 있습니다. 실무에서도 표준 관행입니다.

## 3-2. 모델링 + 평가 (15분)

### LSTM 모델 적용

2층 LSTM 모델을 사용합니다. `LSTM(64, return_sequences=True)` → `Dropout(0.2)` → `LSTM(32)` → `Dropout(0.2)` → `Dense(32, relu)` → `Dense(1)`. 배치 사이즈 256, 최대 100 epoch, Early Stopping 적용합니다.

### 평가 지표: RMSE만으로는 부족한 이유

RMSE의 한계를 이야기해 봅시다.

RMSE는 조기 예측 오류와 지연 예측 오류를 동일하게 취급합니다. "교체하라고 했는데 아직 멀썼"는 조기 예측, 비용은 불필요한 교체입니다. "괜찮다고 했는데 고장 났어"는 지연 예측, 비용은 고장 + 생산 중단 + 안전 사고입니다. 당연히 지연 예측이 훨씬 위험하죠.

그래서 **NASA Scoring Function**을 사용합니다. 비대칭 패널티를 적용합니다. 지연 예측에는 `exp(-d/13) - 1`, 조기 예측에는 `exp(d/10) - 1`. 지연 예측에 더 높은 패널티를 부여하는 거죠.
