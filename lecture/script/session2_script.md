# 섹션 1 — Autoencoder: 정상을 학습해서 이상을 찾는다 (45분)

> 참고 교재: *Hands-On Unsupervised Learning Using Python* (Ankur A. Patel)
> 소요 시간: 문제 제기 3분 + 이론 13분 + 시연 10분 + 실습 19분

## 1-1. 문제 제기 (3분)

Session 1에서 Isolation Forest로 이상 탐지를 해봤습니다. 트리 분할 횟수로 정상과 이상을 구분하는 방식이었죠. 잘 동작했고, 충분히 강력한 도구입니다. 그런데 실제 제조 현장에 한 발 더 들어가면 한 가지 큰 문제가 있습니다.

**라벨 문제**입니다.

상황을 그려 볼게요. 어느 공장에서 수만 개의 정상 제품이 생산되는 동안 결함은 수십 건 나옵니다. 그 수십 건의 결함에 **정확한 라벨**이 붙어 있을 가능성은 얼마나 될까요? 현실은 이렇습니다. "이건 분명 결함인 것 같은데 어떤 종류인지 모르겠다", "이건 정상인가 결함인가 경계가 애매하다", "검사원마다 판정이 다르다". 라벨이 아예 없거나, 있더라도 불확실한 경우가 압도적으로 많습니다.

이게 왜 문제냐. **지도 학습은 라벨을 먹고 자랍니다.** RandomForest든 신경망이든 "이건 정상, 저건 결함"이라는 정답을 보여 줘야 학습이 됩니다. 라벨이 없으면 출발 자체가 안 됩니다. 그렇다고 새로운 결함이 나타날 때마다 사람이 일일이 라벨링하는 것도 비현실적이죠. 비용도 비용이지만, 신제품 라인이 가동되면 라벨이 쌓이기 전에 모델이 필요하거든요.

*Hands-On Unsupervised Learning Using Python* 1장에서 이 약점을 정직하게 짚습니다. "**라벨이 충분히 있으면 지도 학습이 우월하다. 하지만 대부분의 실제 문제에서는 라벨이 부족하다.**" 그리고 Yann LeCun의 유명한 비유를 인용해요. **"지능이 케이크라면, 비지도 학습은 케이크 자체이고, 지도 학습은 그 위의 아이싱이다."** 세상의 대부분의 데이터는 라벨이 없고, 그래서 라벨 없이 학습하는 능력이 본질이라는 메시지죠.

오늘 섹션 1의 핵심 질문은 이렇게 잡고 갑니다. **"라벨 없이, 정상 데이터만 가지고 이상을 탐지할 수 있는가?"**

답은 가능합니다. 핵심 아이디어가 한 줄로 요약됩니다. **"정상만 학습한 모델은 이상을 재현하지 못한다."** 정상 패턴을 압축했다가 복원하는 능력은 키웠지만, 처음 보는 이상 패턴은 복원이 안 되거든요. 그 복원 실패의 크기, 즉 **재구성 오차(reconstruction error)**가 이상 점수가 됩니다.

이걸 가능하게 해 주는 도구가 바로 **Autoencoder**입니다.

## 1-2. 이론 (13분)

### ① 비지도 학습의 프레임워크 (3분)

*Hands-On Unsupervised Learning* 1장과 4장 내용입니다.

먼저 비지도 학습이 뭔지 명확히 하고 넘어가죠. 지도 학습은 입력과 정답 라벨이 쌍으로 있습니다. 모델이 정답을 맞추도록 학습하죠. 반면 비지도 학습은 라벨이 없습니다. 데이터 자체의 구조를 학습합니다. 교재 1장에서 설명하듯이, 비지도 학습은 데이터를 클러스터링하거나 차원을 축소하거나 이상을 탐지하는 등 라벨 없이도 유용한 작업을 수행할 수 있습니다.

비지도 학습이 답하는 질문은 두 가지입니다. "이 데이터는 어떤 구조를 가지고 있는가?" 그리고 "이 샘플은 다른 샘플들과 얼마나 다른가?"

이상 탐지에서의 비지도 학습 전략은 세 단계입니다. 첫째, 정상 데이터만으로 "정상이 어떤 모습인지" 학습합니다. 둘째, 새로운 데이터가 들어왔을 때 "정상과 얼마나 다른지" 측정합니다. 셋째, 차이가 임계값을 넘으면 이상으로 판단합니다.

Session 1에서 배운 PCA, 차원 축소도 비지도 학습의 일종입니다. 고차원 데이터를 저차원으로 압축하면서 중요한 패턴만 남기는 거죠. *Hands-On Unsupervised Learning* 3장에서 다루는 PCA는 데이터의 분산을 최대화하는 축을 찾아 차원을 줄이는 기법입니다. 하지만 PCA는 선형 변환이라 복잡한 패턴을 잡기 어렵습니다.

오늘 배울 Autoencoder는 비선형 차원 축소를 수행합니다. 더 복잡한 패턴을 학습할 수 있죠. 교재 4장에서는 PCA를 포함한 다양한 차원 축소 기법으로 신용카드 사기 탐지 시스템을 구축하는 예제를 다룹니다. 오늘 우리가 하는 실습과 동일한 원리입니다.

### ② Autoencoder 구조와 재구성 오차 (5분)

*Deep Learning with Python* 2판 내용입니다.

Autoencoder는 두 부분으로 구성됩니다.

**인코더**는 입력을 압축합니다. 64차원 입력을 32차원으로, 다시 8차원으로 줄입니다. 이걸 **잠재 공간, latent space**라고 부릅니다. *Deep Learning with Python* 1장에서 설명하는 "표현 학습, representation learning"의 핵심입니다. 데이터를 다른 관점에서 바라보는 것, 더 유용한 형태로 변환하는 것이죠. 교재에서는 이것을 "다단계 정보 증류 과정"이라고 표현합니다. 각 층을 거치면서 중요한 정보만 남고 불필요한 정보는 걸러지는 거죠.

**디코더**는 압축된 표현을 다시 복원합니다. 8차원을 32차원으로, 다시 64차원으로 늘립니다.

손실 함수는 MSE, 평균 제곱 오차입니다. 원본 입력과 복원된 출력이 얼마나 다른가를 측정하죠. 교재 1장에서 설명하듯이, 손실 함수는 모델의 출력이 기대와 얼마나 다른지 측정하는 피드백 신호입니다. 이 피드백을 이용해 옵티마이저가 가중치를 조정하는 것이 학습입니다.

이상 탐지 원리는 이렇습니다. 정상 입력은 인코더가 잘 압축하고 디코더가 잘 복원합니다. 왜냐하면 정상 데이터로만 학습했으니까요. 정상 패턴은 잠재 공간에 잘 표현되어 있어서 재구성 오차가 낮습니다. 하지만 이상 입력은 학습한 적 없는 패턴이니까 인코더가 제대로 압축하지 못하고, 디코더도 제대로 복원하지 못합니다. 재구성 오차가 높게 나오는 거죠.

**재구성 오차 = 이상 점수**입니다.

*Hands-On Unsupervised Learning* 4장에서는 이 원리를 신용카드 사기 탐지에 적용합니다. 28만 건의 거래 중 492건만 사기인 데이터셋에서, PCA로 차원 축소 후 재구성 오차를 계산해서 라벨 없이도 80%의 사기를 탐지해냅니다. 재구성 오차가 가장 높은 상위 350개 거래를 확인했더니 264개가 실제 사기였습니다. 75%의 정밀도를 달성한 거죠.

PyTorch로 구현하면 이렇습니다. 인코더는 Linear 64에서 32, ReLU, Linear 32에서 8. 디코더는 Linear 8에서 32, ReLU, Linear 32에서 64. 간단하죠. `nn.Sequential`이 층들을 순서대로 묶어주는 컨테이너이고, `latent_dim=8`이 잠재 공간의 차원입니다. 이 숫자가 작을수록 압축이 강해지고 이상 탐지에 유리합니다.

잠재 공간 차원, latent_dim은 중요한 하이퍼파라미터입니다. 너무 크면 이상도 잘 재구성되어서 이상 탐지가 안 됩니다. 교재 4장에서 PCA 성분을 30개, 즉 원본 차원 수와 동일하게 설정했을 때 사기 탐지가 전혀 안 되는 걸 보여줍니다. 너무 작으면 정상도 재구성하지 못합니다. 적절한 크기를 찾는 게 관건이고, 이게 곧 실습 과제입니다.

### ③ 임계값 설정: 어디서부터 이상인가 (5분)

*Hands-On Unsupervised Learning* 4장 내용입니다.

재구성 오차를 계산했으면, 어디서부터 이상으로 볼 건지 임계값을 정해야 합니다. 모델을 학습하는 것만큼이나 임계값 설정이 중요합니다. 실무에서는 이 단계에서 가장 많은 시간을 씁니다. 아무리 좋은 모델을 만들어도 임계값을 잘못 정하면 실전에서 소용이 없으니까요.

세 가지 방법이 있습니다.

첫째, **정상 오차의 95번째 백분위수**. 정상 데이터의 재구성 오차를 정렬해서 상위 5%를 이상으로 간주하는 겁니다. "정상 샘플의 5%는 이상으로 오탐되는 것을 허용하겠다"는 의미입니다. 교재 4장에서도 이 방식으로 정상 데이터의 재구성 오차 분포에서 임계값을 설정합니다.

둘째, **평균 + 3시그마**. 3-시그마 규칙이라고도 합니다. 정상 오차의 평균에서 표준편차의 3배를 넘으면 이상으로 보는 거죠. 정규분포를 가정할 때 유효한 방법입니다. 정상 데이터의 99.7%가 평균에서 3시그마 이내에 있다는 통계적 성질을 이용합니다.

셋째, **도메인 전문가 판단**. 실제 현장에서는 전문가가 경험적으로 임계값을 정하기도 합니다. "재구성 오차 0.05 이상이면 작업자에게 알림"처럼 구체적인 숫자를 정하는 거죠. 실무에서 최종 조정은 항상 사람이 합니다.

트레이드오프가 있습니다. 임계값이 낮으면 민감도는 높아지지만 오탐율도 높아집니다. 정상을 이상으로 오탐하는 빈도가 늘어나서 현장 작업자가 알림에 피로를 느끼게 됩니다. "늘 알림이 울리는데 다 괜찮은 거네"라며 알림을 무시하게 되면 오히려 위험합니다. 안전이 중요한 설비에 적합합니다. 반대로 임계값이 높으면 오탐율은 낮아지지만 이상 탐지를 놓칠 수 있습니다. 알림이 울리면 진짜 이상일 확률이 높습니다. 단순 품질 검사에 적합합니다.

교재 4장에서도 강조하듯이, 비지도 학습 시스템의 평가는 지도 학습보다 훨씬 어렵습니다. 알려진 사기 패턴은 잡을 수 있지만, 아직 발견되지 않은 새로운 사기 패턴은 평가할 수 없기 때문입니다. 비지도 학습의 진정한 가치는 바로 이 지점에 있습니다. 알려지지 않은 새로운 이상 패턴을 발견할 수 있는 가능성이 있는 거죠.

## 1-3. Claude Code 시연 (10분)

자, 이론은 여기까지 하고 코드로 직접 확인해 봅시다. 이번 시연의 포인트를 먼저 짚을게요. **모델 구현 자체가 아니라, "재구성 오차가 실제로 이상 신호를 포착하는지"를 데이터로 확인하는 흐름**에 집중해 주세요. Autoencoder는 PyTorch 몇 줄이면 만들 수 있어요. 진짜 어려운 건 결과의 분포를 보고 의미를 읽어내는 부분입니다.

저는 이런 프롬프트로 시작하겠습니다.

```
제조 진동 데이터 이상탐지용 Autoencoder를 만들어줘.
- 정상 데이터: 사인파 기반 합성 신호 (1000개)
- 이상 데이터: 노이즈가 추가된 신호 (50개, 학습에는 사용 안 함)
- PyTorch로 Autoencoder 구현 (input_dim=64, latent_dim=8)
- 정상 데이터로만 학습 (epoch=50)
- 결과: 정상/이상 재구성 오차 분포를 겹쳐서 시각화
- 95th percentile 임계값 표시
```

*(시연 진행)*

코드가 만들어지는 동안 다섯 단계로 흐름을 따라가 보세요.

**1단계, 합성 데이터 생성.** 정상은 깨끗한 사인파에 작은 가우시안 노이즈를 더한 신호, 이상은 같은 사인파에 강한 노이즈와 임의의 스파이크를 섞은 신호입니다. 실제 제조 현장의 정상 진동과 결함 진동을 극단적으로 단순화한 모델이에요. 정상 1000개, 이상 50개를 만들고 이상 50개는 **학습에 절대 들어가지 않습니다**. 평가용으로만 쓰는 거죠.

**2단계, Autoencoder 정의.** PyTorch로 다섯 줄이면 끝납니다. 인코더는 `Linear(64→32) → ReLU → Linear(32→8)`, 디코더는 `Linear(8→32) → ReLU → Linear(32→64)`. 가운데 잠재 공간 8차원이 정보의 병목 역할을 합니다. *Hands-On Unsupervised Learning* 4장에서 PCA 기반 이상탐지를 다룰 때 이 병목의 중요성을 강조하는데, Autoencoder도 같은 원리입니다.

**3단계, 정상 데이터로만 학습.** MSE 손실로 정상 입력과 복원 출력 사이의 거리를 줄이는 방향으로 50 epoch 학습합니다. 핵심은 **이상 데이터는 학습에 들어가지 않는다**는 점이에요. 모델은 "정상이 어떻게 생겼는가"만 알고 있는 상태가 됩니다.

**4단계, 재구성 오차 계산.** 학습이 끝나면 정상 1000개와 이상 50개 각각에 대해 입력과 복원 출력의 MSE를 계산합니다. 정상은 학습한 패턴이라 오차가 낮고, 이상은 처음 보는 패턴이라 오차가 높을 거라는 게 가설이죠.

**5단계, 분포 비교 시각화.** 히스토그램을 겹쳐서 그립니다. 정상 분포가 왼쪽에 몰려 있고, 이상 분포가 오른쪽에 위치하면 성공입니다. 그리고 정상 오차의 95번째 백분위수에 빨간 세로선을 긋습니다. 이게 임계값입니다.

자, 결과가 나왔네요. **정상의 재구성 오차는 평균 0.01 근처에 몰려 있고, 이상은 0.05~0.15 사이에 퍼져 있습니다.** 두 분포가 거의 겹치지 않습니다. 임계값 선이 두 분포 사이에 깔끔하게 위치해 있고, 이 선을 넘은 이상 샘플의 비율이 약 90% 이상이 됩니다.

이게 *Hands-On Unsupervised Learning* 4장의 신용카드 사기 탐지 사례와 정확히 같은 원리예요. 책에서는 28만 건 거래 중 492건만 사기인 데이터로 PCA 재구성 오차 기반 탐지를 만들었는데, 상위 350개 의심 거래 중 264개가 실제 사기였습니다. 정밀도 75%. 라벨 없이도 강력하게 작동한다는 증거였죠.

시연 후 질문을 하나 던지겠습니다. **"latent_dim을 8에서 2로 줄이면 분리가 어떻게 변할까? 반대로 32나 64로 키우면?"**

생각해 보세요. 잠재 공간이 작아지면 압축이 강해집니다. 정상의 핵심 패턴만 살아남고 그 외에는 다 정보가 손실되죠. 이상은 더더욱 복원이 안 되니까 **분리가 더 선명해질 수 있습니다.** 하지만 너무 작으면 정상도 제대로 복원이 안 돼서 오차가 전반적으로 올라가고, 임계값이 모호해집니다.

반대로 latent_dim을 32나 64로 키우면? 잠재 공간이 입력 차원과 같아지니까 사실상 그냥 복사가 가능해집니다. 이상조차도 거의 완벽히 복원해 버리죠. **두 분포가 겹쳐서 분리가 안 됩니다.** 책에서 PCA 성분 수를 원본 차원과 동일하게 설정하면 사기 탐지가 전혀 안 되는 사례가 나오는데, Autoencoder도 똑같습니다.

이 "적절한 압축 강도"를 찾는 게 다음 실습의 핵심 과제입니다.

## 1-4. 실습 (19분)

과제는 `latent_dim`을 변경하며 정상과 이상의 분리를 비교하는 겁니다. 이론에서 배운 "적절한 압축 강도 찾기"를 직접 실험하는 시간입니다.

시작 코드가 제공됩니다. 64 타임스텝 사인파 정상 데이터 1000개, 강한 노이즈 이상 데이터 50개, Autoencoder 클래스, 그리고 `train_and_evaluate` 스켈레톤이 준비되어 있습니다. 스켈레톤 안에는 TODO 주석으로 어떤 부분을 채워야 하는지 표시되어 있으니 그걸 따라가면 됩니다.

각 latent_dim에 대해 95번째 백분위수 임계값으로 이상 탐지율을 측정하세요. `threshold = np.percentile(normal_errors, 95)`로 임계값을 구하고, `(anomaly_errors > threshold).mean()`으로 이상 탐지율을 계산하면 됩니다. 이 값이 높을수록 좋은 겁니다. 이상을 이상으로 잘 탐지한다는 뜻이니까요.

실습 포인트는 세 가지입니다. 첫째, 각 latent_dim에서 정상과 이상의 재구성 오차 분포가 어떻게 달라지는지 그래프로 확인하세요. 둘째, 탐지율이 가장 높은 latent_dim을 찾으세요. 셋째, 압축이 너무 약할 때와 너무 강할 때 각각 어떤 문제가 생기는지 직접 경험해 보세요.

## 2-1. 문제 제기 (3분)

섹션 2 시작하겠습니다.

이번에는 다른 문제를 다룹니다. "이상인가 아닌가"가 아니라 **"얼마나 더 버틸 수 있는가"**를 예측하는 겁니다. 이걸 **RUL, Remaining Useful Life, 잔여 유효 수명**이라고 부릅니다.

유지보수에는 세 가지 전략이 있습니다.

첫째, **고장 후 수리**. 사후보전이라고도 합니다. 고장 나면 고치는 거죠. 비용이 최대이고 위험도 최대입니다. 긴급 수리 비용에 생산 중단까지 더해지니까요. 그런데도 많은 현장이 아직 이 방식입니다.

둘째, **예방 정비**. 예방보전이라고도 합니다. 일정 주기로 교체하는 겁니다. 비용은 중간이지만, 아직 쓸 수 있는 부품을 버리게 됩니다. 예를 들어 30일마다 무조건 교체하면, 실제로는 50일까지 쓸 수 있는 부품을 20일 일찍 버리는 셈이죠. 낭비입니다.

셋째, **예지 정비**. 예지보전이라고도 합니다. RUL 예측을 기반으로 교체 시점을 결정하는 겁니다. 비용이 최소이고 위험도 최저입니다. 필요한 시점에 정확하게 교체하니까요.

예를 들어볼까요. 총 수명 300사이클인 엔진이 있습니다. 사이클 1에서 RUL은 299. 사이클 250에서 RUL은 50, 여기서 교체를 결정합니다. 사이클 300에서 RUL은 0, 고장입니다. RUL 예측이 정확하면 사이클 250에 미리 교체해서 고장을 막을 수 있습니다.

핵심 질문: **현재 센서 데이터만으로 RUL을 예측할 수 있는가?** *ML for Time-Series with Python* 7장에서 다루는 것처럼, 시계열 데이터에서 시간 패턴을 학습하면 가능합니다.

## 2-2. 이론 (13분)

### ① 왜 일반 회귀가 아닌 LSTM인가 (3분)

*ML for Time-Series with Python* 7장 내용입니다.

의문이 드실 겁니다. RUL 예측이면 그냥 선형 회귀나 MLP로 하면 되지 않나?

안 되는 이유는 **순서가 중요하기 때문**입니다.

같은 센서 값이라도 어느 사이클에서 측정됐느냐에 따라 RUL이 완전히 다릅니다. 온도가 300도인데 초반이면 정상일 수 있고, 말기이면 마모 징후일 수 있습니다. 센서 값 자체보다 "센서 값이 어떻게 변해왔는가"가 중요한 거죠. *ML for Time-Series with Python* 2장에서 시계열 분석의 핵심은 "시간에 따른 패턴을 이해하는 것"이라고 강조합니다.

*Deep Learning with Python* 10장에서 이렇게 설명합니다. 밀집 연결 네트워크, 즉 MLP는 입력 데이터를 펼쳐서 처리합니다. 시간이라는 개념이 사라지는 거죠. 하지만 시계열 데이터에서는 시간의 순서가 핵심 정보입니다.

MLP는 각 샘플을 독립적으로 처리해서 순서 정보를 무시합니다. 그냥 특징들만 펼쳐서 넣죠. 하지만 LSTM은 시퀀스를 기억합니다. 시간에 따른 변화 패턴을 학습하는 거죠. 이것이 근본적인 차이입니다.

교재 10장에서 독일 예나 날씨 데이터로 온도 예측 실험을 합니다. MLP 모델은 상식적인 기준선, 즉 "내일 온도는 오늘과 같다"는 예측조차 이기지 못합니다. 하지만 LSTM은 기준선을 넘어서는 성능을 보입니다. 시간 패턴을 학습할 수 있기 때문입니다.

### ② LSTM의 직관 (5분)

*Deep Learning with Python* 2판 내용입니다.

LSTM, Long Short-Term Memory의 핵심은 **무엇을 기억하고 무엇을 버릴지 학습**한다는 겁니다.

일반 RNN에는 기울기 소실 문제가 있습니다. 먼 과거의 정보가 현재에 영향을 주지 못하는 거죠. LSTM은 1997년에 개발된 이래 시계열 딥러닝의 기본 기술로 자리 잡았습니다. *Deep Learning with Python* 1장에서 언급하듯이, LSTM은 1997년에 개발된 이후 거의 변하지 않고 여전히 널리 사용되고 있습니다. 그만큼 설계가 훌륭했다는 뜻이죠.

세 가지 게이트가 있습니다.

**입력 게이트**는 새로운 정보를 얼마나 받아들일지 결정합니다. **망각 게이트**는 기존 정보를 얼마나 잊을지 결정합니다. **출력 게이트**는 현재 상태에서 무엇을 출력할지 결정합니다.

엔진 마모 예로 생각해 보세요. 초반에는 정상 패턴을 유지합니다. 망각 게이트가 이전 정보를 그대로 유지하면서 입력 게이트는 새로운 정상 패턴을 조금씩 받아들이는 거죠. 마모가 진행되면 온도 상승 패턴이 누적됩니다. LSTM이 이 변화를 기억하는 겁니다. 말기에는 누적된 마모 패턴을 바탕으로 RUL이 낮다는 신호를 출력합니다.

슬라이딩 윈도우를 사용합니다. `window_size=30`이면 사이클 1부터 30까지의 센서 데이터, 30곱하기 14 행렬이 LSTM의 입력이 되고, 출력이 RUL 예측값이 됩니다. *Deep Learning with Python* 10장에서 설명하는 `timeseries_dataset_from_array` 함수가 바로 이 슬라이딩 윈도우를 자동으로 생성해 줍니다. 데이터 배열에서 연속된 구간을 잘라서 학습용 샘플을 만드는 거죠.

Keras로 구현하면: `LSTM(64, input_shape=(30, 14))` 다음에 `Dropout(0.2)` 다음에 `Dense(32, relu)` 다음에 `Dense(1)`. `input_shape=(30, 14)`는 30개 사이클 동안 14개 센서값을 의미하고, `return_sequences=False`는 마지막 시점의 출력만 반환한다는 뜻입니다. 마지막 `Dense(1)`은 활성화 함수 없이 숫자를 그대로 출력합니다. 회귀 문제이기 때문입니다. 옵티마이저는 adam, 손실함수는 mse, 메트릭은 mae입니다.

### ③ 과적합 방지: Dropout과 Early Stopping (5분)

*Deep Learning with Python* 5장 내용입니다.

딥러닝에서 가장 큰 적은 과적합입니다. *Deep Learning with Python* 5장에서 "머신러닝의 근본적인 문제는 최적화와 일반화 사이의 긴장"이라고 말합니다. 최적화는 학습 데이터에 최대한 잘 맞추는 것이고, 일반화는 새로운 데이터에도 잘 작동하는 것입니다. 이 둘이 항상 같은 방향으로 가지 않는 게 문제입니다.

과적합이 발생하는 신호를 아는 게 중요합니다. 학습 손실은 계속 감소하는데 검증 손실은 어느 순간 다시 상승하기 시작합니다. 바로 이 시점이 과적합이 시작되는 지점입니다. 교재 5장에서 이것을 "정준 과적합 패턴"이라고 부릅니다. 어떤 모델 타입이든 어떤 데이터셋이든 항상 나타나는 보편적인 패턴입니다.

**Dropout**은 학습 중에 뉴런의 일부를 무작위로 비활성화하는 기법입니다. *Deep Learning with Python* 5장에서 Geoffrey Hinton이 은행에서 영감을 얻었다고 설명합니다. 은행에서 담당 직원들을 무작위로 바꾸면 직원들끼리 공모해서 사기를 칠 수 없듯이, 뉴런도 무작위로 꺼버리면 특정 뉴런에 의존하지 않게 됩니다. 모델이 특정 뉴런에 과의존하지 않게 만들어서 일반화 성능을 높입니다. 비율은 보통 0.2~0.3을 씁니다. 매 배치마다 다른 네트워크를 학습하는 것과 유사한 앙상블 효과가 있습니다. 추론 시에는 자동으로 비활성화됩니다.

**Early Stopping**은 검증 손실이 더 이상 개선되지 않으면 학습을 중단하는 겁니다. `patience=10`이면 10 epoch 동안 검증 손실이 개선되지 않으면 멈춥니다. `restore_best_weights=True`로 최적의 가중치를 복원합니다. `validation_split=0.2`로 학습 데이터의 20%를 검증용으로 분리합니다.

*Deep Learning with Python* 5장에서 권장하는 워크플로우는 이렇습니다. 먼저 과적합이 발생할 수 있는 모델을 만듭니다. 그 다음 정규화 기법을 적용해서 과적합을 제어합니다. 즉, 일부러 과적합을 먼저 만들고 그다음에 해결하는 전략입니다. "완벽한 적합을 찾으려면 먼저 과적합을 해야 한다"는 게 이 장의 핵심 메시지입니다.

이 두 가지를 조합하면 과적합을 효과적으로 방지할 수 있습니다. 다음 시연에서 이걸 직접 확인해 보죠.

## 2-3. Claude Code 시연 (10분)

이번 시연은 코드를 짜는 게 핵심이 아닙니다. 코드는 Claude가 만들어요. 진짜 봐야 할 건 **학습 곡선과 산점도가 들려주는 이야기**입니다. 그 그림에서 무엇이 정상이고 무엇이 경고 신호인지 읽어내는 게 오늘의 훈련 포인트예요.

저는 이런 프롬프트로 시작하겠습니다.

```
NASA Turbofan FD001 데이터로 LSTM RUL 예측을 구현해줘.
- RUL 클리핑: max 125
- window_size=30, 14개 센서
- LSTM(64) -> Dropout(0.2) -> Dense(32) -> Dense(1)
- Early Stopping patience=10
- 학습/검증 손실 곡선 시각화
- 실제 vs 예측 RUL 산점도
```

*(시연 진행)*

시연을 다섯 단계로 따라가 봅시다.

**1단계, 데이터 로드와 RUL 라벨 생성.** FD001은 100개 엔진의 사이클별 센서 데이터입니다. RUL은 각 시점에서 "고장까지 남은 사이클 수"인데, 데이터에 직접 들어 있지 않아요. 각 유닛의 최대 사이클에서 현재 사이클을 빼서 만들어야 합니다. 그리고 125로 클리핑합니다. 왜 125인지는 섹션 3에서 자세히 다루는데, 미리 한 줄로 정리하면 "엔진 초기 구간은 센서 값이 비슷해서 큰 RUL을 구분하는 게 무의미하기 때문"이에요.

**2단계, 슬라이딩 윈도우 시퀀스 구성.** 14개 센서 × 30 사이클 = (30, 14) 형태의 입력 텐서가 한 샘플이 됩니다. Chollet 책 10장에서 소개하는 `timeseries_dataset_from_array` 함수가 이 자르기를 자동으로 해 줘요. 슬라이딩 윈도우 크기, 스트라이드, 배치 사이즈 같은 파라미터를 한 번에 받습니다.

**3단계, 모델 정의와 학습.** `LSTM(64) → Dropout(0.2) → Dense(32, relu) → Dense(1)` 구조. 회귀니까 마지막 Dense에는 활성화 함수가 없어요. 옵티마이저는 adam, 손실은 mse, 모니터링용 메트릭은 mae입니다. Early Stopping을 `patience=10`, `restore_best_weights=True`로 걸어 둡니다.

**4단계, 학습 곡선 읽기.** 화면을 보세요. 학습 손실은 계속 떨어지는데, **검증 손실은 어느 시점에서 꺾이고 다시 올라가기 시작**합니다. 그 지점이 과적합의 시작이에요. *Deep Learning with Python* 5장에서 이걸 **"정준 과적합 패턴, the canonical overfitting pattern"** 이라고 부릅니다. 어느 모델, 어느 데이터셋에서도 보편적으로 나타나는 모양이라는 뜻이죠. Early Stopping은 검증 손실이 최저였던 시점의 가중치를 복원해서, 우리에게 "과적합 직전의 최적 모델"을 돌려줍니다.

**5단계, 산점도로 예측 품질 확인.** x축이 실제 RUL, y축이 예측 RUL입니다. 완벽한 모델이면 모든 점이 대각선 위에 있어야 하죠. 실제로 보면 대각선을 따라가긴 하는데 **RUL이 큰 구간, 즉 초기 마모 단계에서 예측값이 실제보다 낮게 나옵니다.** 모델이 보수적으로 예측하는 거예요. 이건 두 가지 이유가 겹친 결과입니다. 첫째, 초기 구간의 센서 값이 RUL 200, 250, 300에서 거의 비슷해서 모델이 구분하지 못합니다. 둘째, 우리가 125로 클리핑했기 때문에 모델이 큰 RUL을 학습해 본 적이 없어요. 그래서 실제 RUL이 300이어도 모델은 125 이하로 답하는 경향이 생깁니다. 이건 **버그가 아니라 의도된 설계**입니다.

시연 후 질문을 하나 던지겠습니다. **"RMSE는 낮은데 실제로 고장 알림이 잘 작동하지 않는다면 어디가 문제인가? 비대칭 평가 지표가 왜 필요한가?"**

핵심은 RMSE가 **조기 예측과 지연 예측을 똑같이 취급**한다는 점입니다. 두 케이스를 비교해 볼게요. 실제 RUL이 10인데 모델이 20이라고 예측 → **10사이클 조기 예측.** 결과는 불필요한 교체. 비용이 들지만 설비는 안전합니다. 반대로 실제 RUL이 10인데 모델이 0이라고 예측 → **10사이클 지연 예측.** 결과는 고장 발생, 생산 중단, 안전 사고. 두 오차의 크기는 같은 10인데, 비즈니스 충격은 완전히 다릅니다.

RMSE는 둘을 같은 크기의 오차로 봅니다. 그래서 NASA Scoring Function 같은 **비대칭 평가 지표**가 필요한 거예요. 지연 예측에 더 큰 패널티를 부여해서 "안전 측면에서 모델이 어느 정도로 신뢰할 만한가"를 측정합니다. 섹션 3에서 직접 구현해 봅니다.

## 3-1. 데이터 확인 + RUL 라벨 설계 (10분)

마지막 섹션입니다. 실제 데이터셋으로 지금까지 배운 걸 종합해서 적용해 보겠습니다.

**NASA Turbofan 데이터셋**은 항공 엔진의 센서 데이터로, RUL 예측 연구의 벤치마크입니다. *ML for Time-Series with Python*에서 시계열 분석의 실제 사례로 활용하는 데이터셋 중 하나입니다. NASA에서 공개한 이 데이터는 전 세계 수백 편의 논문에서 사용되었습니다.

네 가지 서브셋이 있습니다. FD001은 단일 운전 조건, 단일 고장 모드입니다. 가장 단순한 케이스죠. FD002는 6가지 조건, 단일 고장 모드. FD003은 단일 조건, 2가지 고장 모드. FD004는 6가지 조건, 2가지 고장 모드입니다. 오늘은 FD001을 사용합니다. 복잡도가 낮아서 기본 개념을 익히기에 적합합니다.

열 구성은: 유닛 번호, 사이클 번호, 운전 조건 3개, 센서 값 21개입니다. 총 26개 열이죠. FD001은 100개 엔진의 고장까지의 전체 이력을 담고 있고, 평균 수명은 약 204사이클입니다. 가장 단명한 엔진은 126사이클, 가장 장수한 엔진은 361사이클까지 갑니다. 참고로 이 실습용 사본은 원본 C-MAPSS를 5사이클 간격으로 서브샘플한 버전이라, 행 수나 수명 수치가 공식 데이터셋과 조금 다릅니다.

먼저 Claude에게 EDA를 요청해 보겠습니다. 데이터 형태, 결측치, 유닛 수, 사이클 범위, 센서별 시계열 플롯, 그리고 저분산 센서, 표준편차가 0.01 미만인 센서를 식별합니다. 센서 21개 중에서 변동이 거의 없는 센서를 골라내는 게 중요합니다. 정보가 없는 센서는 모델에 도움이 안 되거든요. *ML for Time-Series with Python* 2장에서 변수 이해의 첫 단계가 요약 통계와 분포 확인이라고 합니다.

*(시연 진행)*

EDA 결과를 보면 몇 가지 핵심 사실이 나옵니다. 첫째, 결측치는 없습니다. 둘째, FD001에는 100개 유닛이 있습니다. 셋째, 21개 센서 중 6개가 거의 상수입니다. 표준편차가 0.01 미만이에요. 이 센서들은 모든 엔진에서 거의 동일한 값을 유지하므로 RUL 예측에 도움이 되지 않습니다.

21개 센서에서 저분산 센서를 제외한 14개만 사용합니다. 이것이 실습 코드에서 `sensor_cols`에 14개 센서만 포함하는 이유입니다. 센서 선택은 모델 성능에 직접적인 영향을 미칩니다. 불필요한 센서를 포함하면 노이즈가 추가되어 성능이 떨어집니다.

### RUL 클리핑: 왜 125로 자르는가

중요한 전처리가 하나 있습니다. RUL 클리핑입니다.

엔진 초기, 사이클 1~100 정도의 센서 데이터는 모두 "정상"입니다. RUL이 250이든 300이든 350이든 센서 데이터가 비슷하죠. 이걸 다 학습시키면 모델이 초기 구간에서 불필요하게 노력하게 됩니다. 서로 다른 RUL 라벨을 구분하려고 헛수고를 하는 거죠. 센서 값이 비슷한데 라벨만 다르니까 모델이 혼란스러워집니다.

그래서 RUL 상한을 125로 자릅니다. `RUL = (max_cycle - cycle).clip(upper=125)`. 125 이상은 모두 125로 처리하는 거죠. 이렇게 하면 모델이 의미 있는 예측 범위에 집중할 수 있습니다. 실무에서도 표준 관행입니다.

코드로는 이렇습니다. 먼저 각 유닛의 최대 사이클을 구하고, 원본 데이터에 병합한 다음, RUL을 계산하면서 125로 클리핑합니다. 세 줄이면 됩니다.

*ML for Time-Series with Python* 2장에서 설명하는 것처럼, 시계열 분석에서는 데이터의 특성을 먼저 이해하는 게 중요합니다. NASA Turbofan 데이터의 대부분 센서는 엔진 마모가 진행됨에 따라 추세를 보입니다. 통계적 특성이 시간에 따라 변하는 비정상 시계열이죠. 이것이 슬라이딩 윈도우와 LSTM 접근이 유리한 이유입니다.

### EDA 실습 결과 해석

실제 EDA를 돌려본 결과를 같이 확인해 보겠습니다. FD001의 센서-RUL 상관관계를 보면 sensor_11이 -0.70으로 가장 강한 음의 상관을 보입니다. RUL이 감소할수록, 즉 고장에 가까워질수록 이 센서 값이 증가하는 거죠. 반대로 sensor_12는 +0.67, sensor_7은 +0.65로 양의 상관을 보입니다. 정상일 때 높다가 노화가 진행되면 감소하는 센서들입니다.

RUL 구간별로 센서 평균값을 비교해보면 명확한 패턴이 보입니다. RUL이 0에서 50, 즉 고장 임박 구간에서는 sensor_9가 9079, sensor_14가 8153으로 가장 높습니다. 반면 RUL이 300 이상인 초기 구간에서는 sensor_9가 9056, sensor_14가 8137로 상대적으로 낮습니다. RUL이 125 이하로 떨어지면 센서 변화가 가속화되는 "급격한 성능 저하 구간"이 시작됩니다. RUL이 200 이상인 구간에서는 센서 값이 거의 일정합니다. 이게 바로 RUL 클리핑(125)의 실증적 근거입니다. 데이터가 직접 말해주는 거죠.

이 상관관계 정보는 노트북 `h2_autoencoder_anomaly.ipynb`와 `h2_lstm_rul_prediction.ipynb`에서 시각화로 확인할 수 있습니다. 엔진 #1의 노화 패턴을 보면 sensor_12는 시간이 지날수록 서서히 감소하다가 말기에 급격히 떨어집니다. sensor_11은 반대로 말기에 급격히 상승합니다.

## 3-2. 모델링 + 평가 (15분)

### LSTM 모델 적용

섹션 2에서 만든 LSTM 구조를 Turbofan 데이터에 적용합니다. 2층 LSTM 모델을 사용합니다. `LSTM(64, return_sequences=True)` 다음에 `Dropout(0.2)` 다음에 `LSTM(32)` 다음에 `Dropout(0.2)` 다음에 `Dense(32, relu)` 다음에 `Dense(1)`. 배치 사이즈 256, 최대 100 epoch, Early Stopping 적용합니다.

모델을 학습시킨 후에는 두 가지 시각화를 확인합니다. 첫째, 학습/검증 손실 곡선입니다. 과적합이 발생하는지, Early Stopping이 적절한 시점에 작동하는지 봅니다. 둘째, 실제 RUL과 예측 RUL의 산점도입니다. 대각선에 가까울수록 예측이 정확한 겁니다.

여기서 핵심은 모델 구조가 아니라 **평가 지표를 어떻게 해석하느냐**입니다.

### 평가 지표: RMSE만으로는 부족한 이유

RMSE의 한계를 이야기해 봅시다.

RMSE는 조기 예측 오류와 지연 예측 오류를 동일하게 취급합니다. "교체하라고 했는데 아직 멀었어"는 조기 예측, 비용은 불필요한 교체입니다. "괜찮다고 했는데 고장 났어"는 지연 예측, 비용은 고장에 생산 중단에 안전 사고까지입니다. 당연히 지연 예측이 훨씬 위험하죠.

구체적인 예를 들어볼까요. 실제 RUL이 10인데 예측이 20이면, 10사이클 일찍 교체하게 됩니다. 교체 비용은 들지만 설비는 안전합니다. 반대로 실제 RUL이 10인데 예측이 0이면, 고장 날 때까지 기다리게 됩니다. 설비가 고장 나고 생산이 중단되고 안전 사고가 발생할 수 있습니다. 두 오류 모두 크기가 10인데, 결과는 완전히 다릅니다.

그래서 **NASA Scoring Function**을 사용합니다. 비대칭 패널티를 적용합니다. 지연 예측에는 `exp(-d/13) - 1`, 조기 예측에는 `exp(d/10) - 1`. d가 음수면 지연 예측인데, 지연 예측에 더 높은 패널티를 부여하는 거죠. 지수 함수의 기울기가 달라서, 같은 오차 크기라도 지연 예측의 패널티가 조기 예측보다 훨씬 큽니다.

이걸 코드로 구현하면 `nasa_score` 함수가 됩니다. 예측값에서 실제값을 뺀 d를 계산하고, d가 음수면 지연 예측으로 분류해서 더 높은 패널티를 줍니다. 최종 점수는 모든 샘플의 패널티 합입니다. 낮을수록 좋습니다.

이 비대칭 평가 지표가 제조 현장에서 RUL 예측 모델을 실제로 사용할 때 핵심적인 역할을 합니다. RMSE만 보면 안 되는 이유를 꼭 기억하세요. 모델을 선택할 때는 RMSE와 NASA Score를 함께 봐야 합니다.

### 실습 모델 결과 해석

실제로 PyTorch로 구현한 Autoencoder와 LSTM 결과를 같이 보겠습니다.

**Autoencoder 이상 탐지 결과**: 정상 구간, RUL이 125 초과인 데이터로만 Autoencoder를 학습했습니다. 정상 데이터의 재구성 오차 평균이 0.009인데 비해, 노화 구간의 재구성 오차 평균은 0.021로 약 2.2배 높습니다. 임계값을 정상 데이터의 95퍼센타일인 0.0176으로 설정했을 때, 정상 데이터의 95%를 정상으로 올바르게 분류합니다. ROC-AUC가 0.67로, 정상 데이터만 학습했음에도 노화 구간을 재구성 오차로 구분해냅니다. Latent space를 PCA로 시각화하면 건강 상태와 노화 상태가 분리되는 패턴이 확인됩니다.

**LSTM RUL 예측 결과**: 2층 LSTM, hidden_size 64, Dropout 0.2로 학습했습니다. 검증 RMSE는 32 cycles, epoch 29에서 Early Stopping이 발동했습니다. Test RMSE는 52 cycles입니다. 학습된 모델의 예측값 표준편차가 0에 가까운 걸 보면, 30 epoch로는 충분히 학습되지 않았다는 걸 알 수 있습니다. 이건 교육적으로 아주 좋은 사례입니다. 모델이 수렴하지 않은 이유, 학습률이나 epoch 수를 어떻게 조정해야 하는지 생각해 보세요.

**Dropout 실험**: Dropout을 0.0에서 0.5까지 변화시키면서 비교했습니다. Dropout 0.0에서 Test RMSE가 51.47, Dropout 0.2에서 51.25로 약간 개선됩니다. 하지만 Dropout 0.5에서는 54.06으로 악화됩니다. 너무 많은 노드를 꺼버리면 학습이 불충분해지는 거죠. *Deep Learning with Python*에서 Dropout은 "은행의 모든 직원이 동시에 이직하면 안 된다"는 사기 방지 시스템에서 영감받았다고 설명합니다. 적절한 비율이 중요합니다.

### Autoencoder + LSTM 통합 관점

마지막으로 두 모델을 어떻게 통합하는지 개념을 설명하겠습니다. Autoencoder로 이상 점수를 계산하고, LSTM으로 RUL을 예측해서 이상 경고와 RUL 예측을 동시에 출력하는 시스템입니다.

센서 데이터가 들어오면 먼저 Autoencoder가 재구성 오차를 계산합니다. 임계값을 넘으면 즉시 이상 경보를 발생시킵니다. "이 센서 패턴은 정상이 아닙니다"라고 알려주는 거죠. 정상 범위면 LSTM이 RUL을 예측합니다. RUL이 30 이하면 교체 권고를 내보냅니다. "이 엔진은 앞으로 30사이클 이내에 고장날 가능성이 높습니다"라고 예측하는 거죠.

이 통합 파이프라인이 실제 현장에서 가장 강력한 접근입니다. 이상 탐지와 수명 예측을 동시에 수행해서 예지 보전의 두 가지 핵심 질문에 모두 답할 수 있습니다. "지금 이상한가?"와 "얼마나 더 쓸 수 있는가?" 말입니다. 이건 자유 실습에서 심화 과제로 다룹니다.

### 심화 — Autoencoder + LSTM 통합 파이프라인

가장 흥미로운 과제입니다. 섹션 1의 Autoencoder와 섹션 2의 LSTM을 **하나의 시스템으로 통합**합니다.

핵심 아이디어. Autoencoder의 재구성 오차는 "지금 이 윈도우가 얼마나 비정상적인가"를 알려주는 신호죠. 이걸 그냥 별도 출력으로 두지 말고 LSTM의 **추가 입력 특징**으로 합쳐 봅시다.

1. 섹션 1에서 만든 Autoencoder를 FD001 정상 구간(RUL > 125)으로만 학습시킵니다.
2. 모든 윈도우에 대해 재구성 오차를 계산합니다.
3. 기존 14개 센서 + 재구성 오차 = 15차원 특징을 만듭니다.
4. 이 15차원 특징으로 LSTM RUL 예측 모델을 학습합니다.
5. 14차원 베이스라인과 15차원 통합 모델의 RMSE/NASA Score를 비교합니다.

가설은 명확합니다. **재구성 오차가 노화의 조기 신호 역할을 해 줘서 LSTM이 RUL을 더 정확하게 추정할 것**이라는 거죠. 정말 그런지 데이터로 검증하는 거예요.

제출물: **두 모델의 지표 비교 + 재구성 오차가 노화 사이클을 따라 어떻게 변하는지 시계열 그래프 + "통합이 도움이 되었는가, 그 이유는?" 한 문단**.
