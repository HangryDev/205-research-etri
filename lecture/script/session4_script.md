# 섹션 1 — Diffusion Model: 노이즈에서 결함 이미지를 꺼내는 법 (45분)

> 참고 교재: *Generative Deep Learning, 2nd Edition* (David Foster)
> 소요 시간: 문제 제기 3분 + 이론 13분 + 시연 10분 + 실습 19분

---

## 1-1. 문제 제기 (3분)

제조 결함 탐지 모델을 학습하려면 어떤 데이터가 필요할까요?

정상 이미지는 수만 장 쉽게 수집할 수 있습니다. 스크래치 결함은 수백 장 정도 있고요. 버블 결함은 수십 장. 균열 결함은 아예 불충분해서 학습이 불가능합니다.

Session 1에서는 SMOTE로 데이터 불균형을 해결했습니다. 하지만 **이미지에 SMOTE를 적용하면** 어떻게 될까요? 두 이미지의 픽셀을 보간하면 의미 없는 흐릿한 혼합 이미지가 나옵니다. 스크래치 반, 버블 반인 이미지는 아무 의미가 없죠.

필요한 건 **실제처럼 보이는 새로운 결함 이미지**입니다.

핵심 질문: **없는 결함 이미지를 실제처럼 생성할 수 있는가?**

---

## 1-2. 이론 (13분)

### STEP 01 Diffusion의 직관: 노이즈 추가 → 노이즈 제거 (5분)

*Generative Deep Learning* 2판 내용입니다.

Diffusion Model의 핵심 직관은 이겁니다. **모델은 "이미지에서 노이즈를 제거하면 어떤 이미지가 나오는가"를 학습합니다.**

순방향 과정, 노이즈 추가를 먼저 보겠습니다. 원본 이미지에 점진적으로 노이즈를 추가합니다. 흐려지죠. 이걸 T번, 보통 1000번 반복하면 순수한 노이즈가 됩니다.

역방향 과정, 노이즈 제거가 핵심입니다. 순수 노이즈에서 시작해서 점진적으로 노이즈를 제거합니다. 매 단계에서 모델이 "여기서 노이즈를 얼마나 제거해야 하나"를 예측하는 거죠. 이 과정을 반복하면 완전히 새로운 이미지가 생성됩니다.

이게 왜 가능한가? 모델이 학습할 때 수많은 이미지의 노이즈 제거 과정을 학습했기 때문입니다. 한 번도 본 적 없는 새로운 이미지도 순수 노이즈에서 만들어낼 수 있는 거죠.

### STEP 02 Fine-tuning 전략: 적은 데이터로 특정 스타일 학습 (5분)

*Hands-On Generative AI* 내용입니다.

Stable Diffusion 같은 사전 학습 모델은 일반적인 이미지는 잘 생성합니다. 하지만 제조 결함 같은 특정 도메인 이미지는 잘 생성하지 못합니다.

그래서 **Fine-tuning**이 필요합니다. 세 가지 전략이 있습니다.

**DreamBooth**: 3~20장의 이미지로 전체 모델을 업데이트합니다. 수십 분이 걸리지만 가장 정확합니다.

**LoRA**: 10~100장으로 부분 파라미터만 추가합니다. 몇 분이면 되고 효율적입니다.

**Textual Inversion**: 5~10장으로 텍스트 임베딩만 학습합니다. 몇 분이면 되지만 제어력이 제한적입니다.

**LoRA의 직관**을 이해해 봅시다. 원본 가중치는 그대로 고정합니다. 거기에 두 개의 작은 행렬 곱을 추가하는 거죠. delta_W = A × B. 전체 파라미터의 0.1~1%만 학습합니다. 10장의 결함 이미지로도 특정 결함 패턴을 학습할 수 있습니다.

### STEP 03 생성 품질 평가: 무엇이 좋은 이미지인가 (3분)

*Practical Synthetic Data Generation* 1장 내용입니다.

생성된 이미지가 좋은지 어떻게 평가할까요?

**시각적 검사**가 가장 기본입니다. 하지만 주관적이죠.

**SSIM**, Structural Similarity Index는 구조적 유사성을 측정합니다. 0에서 1 사이의 값이고, 실무 기준은 0.7 이상입니다.

**FID**, Frechet Inception Distance는 생성 이미지 분포와 실제 이미지 분포 사이의 거리를 측정합니다. 낮을수록 좋습니다. 10 미만이면 우수, 50 미만이면 양호합니다. 다만 최소 수백 장이 필요합니다.

---

## 1-3. Claude Code 시연 (10분)

시연 포인트: Fine-tuning 학습 과정보다 **생성 결과의 품질을 평가하는 방법**에 집중해 주세요.

```
Stable Diffusion으로 세 가지 결함 이미지를 생성해줘.
- 금속 표면의 스크래치
- 도장 표면의 버블
- 세라믹의 균열
- num_inference_steps=50, 유형당 3장
- 3x3 그리드로 시각화
- SSIM 계산
- GPU 없으면 CPU 대체
```

*(시연 진행)*

세 가지 결함 유형별로 생성된 이미지를 3x3 그리드로 확인합니다. SSIM 점수도 같이 표시됩니다.

시연 후 질문: "Mode collapse는 왜 발생하며 어떻게 해결하는가?"

답: Mode collapse는 모델이 다양한 패턴을 학습하지 못하고 비슷한 이미지만 계속 생성하는 현상입니다. 원인은 학습 데이터 다양성 부족, 학습률 설정, 모델 용량 부족 등입니다. 해결은 데이터 증강, 다양한 프롬프트 사용, LoRA 가중치 조정 등입니다.

## 1-4. 실습 (19분)

과제: `num_inference_steps`를 변경하며 생성 속도와 이미지 품질의 트레이드오프를 분석합니다.

20, 50, 100 세 가지로 실험합니다.

시작 코드가 제공됩니다. StableDiffusionPipeline, GPU/CPU 감지, 세 가지 단계 설정, SSIM 점수, 시간 측정까지 준비되어 있습니다. GPU가 없는 환경에서는 대체 합성 방식을 사용합니다.

실습 주의사항: Fine-tuning 자체는 GPU에서 수십 분~수 시간이 소요됩니다. 이번 실습에서는 사전 준비된 가중치를 로드해서 생성 결과만 실험합니다.

# 섹션 2 — Defect Generation: 프롬프트로 결함을 설계한다 (45분)

> 참고 교재: *Hands-On Generative AI* Ch.2 / *Generative Deep Learning* 2판 Ch.4
> 소요 시간: 문제 제기 3분 + 이론 13분 + 시연 10분 + 실습 19분

---

## 2-1. 문제 제기 (3분)

섹션 1에서 결함 이미지를 생성할 수 있었습니다. 하지만 한 가지 문제가 있습니다.

섹션 1은 무작위 스크래치를 생성합니다. 위치, 깊이, 방향을 제어할 수 없습니다.

제조 현장에서는 **특정 조건의 결함**이 필요합니다. "스테인리스 스틸 표면 왼쪽 가장자리에 5mm 깊이의 수직 스크래치, 고해상도." 이렇게 정확하게 지정하고 싶은 거죠.

핵심 질문: **생성된 결함을 정확하게 제어할 수 있는가?**

---

## 2-2. 이론 (13분)

### STEP 01 조건부 생성의 직관: CGAN에서 Diffusion으로 (4분)

*Generative Deep Learning* 2판 4장 내용입니다.

조건부 생성, 원하는 조건에 맞는 이미지를 만드는 기술의 진화를 보겠습니다.

**CGAN**, 2014년에 나온 Conditional GAN은 클래스 레이블을 조건으로 사용합니다. "스크래치 생성해"라고 하면 스크래치를 만들지만, 제어력이 제한적입니다.

**Text-to-Image Diffusion**, 2022년 이후 기술은 텍스트 문장 전체를 조건으로 사용합니다. "스테인리스 스틸 표면 왼쪽에 깊은 수직 스크래치"라고 하면 그대로 만들어 줍니다. 언어를 통한 **무한한 제어력**이 가능해진 거죠.

핵심 아이디어는 동일합니다. 조건부 생성. 다만 조건을 표현하는 방식이 클래스 레이블에서 자연어로 진화한 겁니다.

### STEP 02 CLIP: 텍스트와 이미지를 연결하는 다리 (5분)

*Hands-On Generative AI* 2장 내용입니다.

Diffusion 모델이 텍스트 프롬프트를 이해하려면 텍스트와 이미지를 연결하는 다리가 필요합니다. 그게 **CLIP**입니다.

CLIP, Contrastive Language-Image Pre-training은 수십억 개의 인터넷 이미지-텍스트 쌍으로 학습했습니다. 핵심 원리는: **비슷한 의미의 텍스트와 이미지는 벡터 공간에서 가까이 위치한다**는 겁니다.

"빨간 자동차"라는 텍스트와 실제 빨간 자동차 사진의 벡터가 가깝게 만들어지는 거죠.

이 CLIP 덕분에 Diffusion 모델이 프롬프트를 "이해"할 수 있게 되었습니다. CLIP 임베딩 코드로 보면, CLIPTokenizer가 프롬프트를 토큰으로 나누고, CLIPTextModel이 임베딩 벡터를 만듭니다. 형태는 [1, 토큰 수, 512]입니다.

### STEP 03 제조 결함 프롬프트 설계 원칙 (4분)

프롬프트 설계가 생성 결과에 직접적인 영향을 미칩니다. 제조 결함 프롬프트 설계 공식을 소개합니다.

**[결함 유형] + [위치] + [심각도] + [재질/배경] + [촬영 스타일]**

좋은 프롬프트 예시: "스테인리스 스틸 표면 왼쪽 가장자리에 깊은 수직 스크래치 결함, 산업용 품질 검사 사진, 고해상도, 스튜디오 조명."

나쁜 프롬프트 예시: "결함." 너무 모호하죠. "매우 매우 나쁜 결함 심각하게 손상됨." 강조를 반복해도 효과가 없습니다.

그리고 **네거티브 프롬프트**도 중요합니다. 원하지 않는 요소를 제거하는 거죠. "흐릿함, 낮은 품질, 만화, 삽화, 텍스트, 워터마크" 같은 키워드를 넣습니다. 5~7개 키워드가 적절합니다. 너무 많이 넣으면 혼란으로 오히려 품질이 저하됩니다.

시연 포인트: **프롬프트 설계가 생성 결과에 미치는 영향**을 직접 비교하는 데 집중해 주세요.


답: Diffusion 모델은 시작 노이즈가 무작위이기 때문에 매번 다른 결과가 나옵니다. `seed`를 고정하면 동일한 프롬프트로 항상 같은 결과를 얻을 수 있습니다.

# 섹션 3 — 합성 데이터 고도화: 만든 데이터가 실제로 도움이 되는가 (50분)

> 참고 교재: *Practical Synthetic Data Generation* (Khaled El Emam et al.)
> 소요 시간: 핵심 개념 10분 + 시연 15분 + 자유 실습 25분

---

## 3-1. 핵심 개념 + 실험 설계 (10분)

마지막 섹션입니다. 가장 중요한 질문을 다룹니다.

**만든 합성 데이터가 실제로 도움이 되는가?**

### 합성 데이터의 기회와 위험

합성 데이터가 효과적일 때가 있습니다. 소수 클래스, 즉 결함을 보강하면 Recall이 향상됩니다. 결함 유형 다양성이 늘어나면 일반화 성능도 향상됩니다. 라벨링 비용 절감으로 대규모 학습이 가능해집니다.

하지만 합성 데이터가 해로울 때도 있습니다. 품질이 낮으면 노이즈를 주입하는 겁이 됩니다. 합성 데이터 비율이 너무 높으면 실제 데이터 분포에서 벗어납니다. 도메인 불일치, 색상이나 텍스처 차이도 문제가 됩니다.

실패 시나리오를 생각해 보세요. 합성 스크래치는 완벽하게 직선이고 깊이가 균일합니다. 하지만 실제 스크래치는 불규칙한 방향과 깊이 변화가 있습니다. 합성 데이터로만 학습하면 실제 스크래치를 탐지하지 못할 수 있습니다.

### 실험 설계: 무엇을 비교할 것인가

핵심 질문은 "얼마나 혼합할 것인가?"입니다.

실험은 이렇게 설계합니다. 실제 결함 50장, 합성 결함 200장, 테스트셋 20장. 테스트셋은 절대 학습에 사용하지 않습니다.

네 가지 데이터 구성을 비교합니다. 실제 50장만 기준선. 1:1, 실제 대 합성. 1:3, 실제 대 합성. 합성 200장만 극단적.

평가는 F1, Recall, Precision으로 합니다. Session 1에서 배운 불균형 평가 지표가 여기서 다시 쓰입니다.

## 3-2. Claude Code 시연 (15분)

시연 포인트: 코드 작성보다 **결과 수치를 보고 Claude에게 "왜 이런 결과가 나왔는지" 질문하는 데** 집중해 주세요.

```
합성 데이터 추가 효과를 비교해줘.

- Confusion Matrix와 F1/Recall/Precision 막대 그래프
- 합성 비율 대 메트릭 라인 그래프
```

*(시연 진행)*

결과를 보면 합성 데이터를 추가했더니 Recall은 올라갔는데 Precision은 내려간 걸 볼 수 있습니다.

시연 후 해석 질문: "합성 데이터를 추가했더니 Recall은 올라갔는데 Precision은 내려갔어. 왜 그런지, 어떤 지표를 신뢰해야 하는가?"

답: 합성 데이터가 더 다양한 결함 패턴을 제공해서 모델이 더 많은 결함을 잡아내기 때문에 Recall이 올라갑니다. 하지만 합성 결함이 실제 결함과 완전히 같지 않기 때문에 오탐도 증가해서 Precision이 내려가는 겁니다. 제조업에서는 Session 1에서 배운 것처럼 Recall이 핵심이므로, 적절한 합성 비율에서 Recall이 크게 향상되는 지점을 찾는 게 중요합니다.
